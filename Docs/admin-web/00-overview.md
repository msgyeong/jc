# 관리자 웹 - 개요

본 문서는 **관리자 웹**(Web 콘솔)의 목적, 범위, 사용자(역할), 정책을 정의한다.

- 용어 기준: `../contributing.md`
- 앱 설계 기준 문서: `../prd/08-design.md`
- 보안/권한 기준 문서: `../prd/04-non-functional.md`
- 데이터/정책 기준 문서: `../prd/05-data-policy.md`

---

## 1. 목적

- **최고 관리자**가 관리자 웹에서 회원 승인/회원 관리/권한 관리를 수행한다.
- 앱의 권한 기능(공지사항 작성, 일정 등록 등)은 **관리자 웹에서 부여한 권한**을
  근거로 한다.
- 관리자 웹은 앱 기능의 “권한/데이터 기준(서버 기준)”을 운영하기 위한 콘솔이다.

---

## 2. 사용자(역할)

- **최고 관리자**
  - 예: `manager_###`
  - 관리자 웹에 로그인하여 전체 권한을 수행한다.

---

## 3. 범위

- **In Scope (MVP)**
  - 최고 관리자 로그인
  - 관리자 계정 보안(비밀번호 생성/변경 등) 정책 적용
  - 회원 가입 승인/거절 및 상태 관리
  - 회원 정보 조회/수정(관리자 전용 필드 포함)
  - 직책/권한 부여·회수
  - 게시판/공지/일정 관리(조회/등록/수정/삭제 등, PRD 정의 범위 내)

- **Out of Scope (추후 확장)**
  - 통계/분석 대시보드(대규모 리포트)

---

## 4. 기본 정책

- 관리자 웹은 **최고 관리자만 접근 가능**하다.
- 관리자 웹에서 변경된 권한/회원 상태는 앱에 즉시 반영되어야 한다.
- 문서의 역할/권한 표기는 `관리자 웹`, `최고 관리자`, `앱 내 관리자(직책 기반)`만
  사용한다.

---

## 5. 보안/권한 정책(요약)

- 최고 관리자 인증이 완료되지 않으면 모든 관리자 웹 화면 접근을 차단한다.
- 관리자 웹에서 비밀번호 생성 및 변경을 통해 관리자 계정 보안을
  생성/유지/관리한다(`prd/04-non-functional.md` 기준).
- 앱에서의 권한은 **관리자 웹에서 부여된 권한**을 기준으로만 판단한다.
  - 권한이 부여되지 않은 사용자는 앱에서 권한 기능에 접근할 수 없다.
  - 권한 정보는 UI/API/데이터 구조 상에서 추측 가능해서는 안 된다.

---

## 6. 기술/운영 메모(요약)

- 관리자 웹은 앱과 동일한 Supabase 백엔드를 사용한다(`prd/08-design.md` 기준).
- 변경 이력(누가/언제/무엇을 변경했는지)은 운영 관점에서 중요하므로,
  가능한 한 감사 로그를 남기는 구조를 권장한다(구현 범위는 별도 확정).
