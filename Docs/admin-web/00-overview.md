# 관리자 웹 PRD - 개요

> 최종 수정: 2026-03-09 | 전면 재작성 (사장님 UI 명세서 기반)
> ⚠️ 기존 admin-web 문서/코드를 전부 폐기하고 이 문서를 새 기준으로 사용

---

## 1. 목적

관리자 웹은 **최고 관리자가 회원/콘텐츠/권한을 통합 관리하는 운영 콘솔**이다.

핵심 가치:
- 입회원서 디지털화 → 회원관리카드 자동 생성/관리 (R-01)
- 직책/권한의 중앙 관리 → 앱 즉시 반영 (R-03)
- 참석 투표 현황 관리 → 성원/비용 파악 (R-02)
- 배너 광고 관리 → 회원 사업장 홍보 승인 (R-05)

---

## 2. 기술 스택

| 항목 | 기술 |
|------|------|
| 프론트엔드 | HTML, CSS, Vanilla JavaScript (SPA) |
| 백엔드 | Node.js Express (기존 API 서버 공유) |
| 데이터베이스 | Railway PostgreSQL (앱과 동일) |
| 인증 | JWT (admin 전용 토큰) |
| 배포 | 기존 Docker 컨테이너 내 `/admin/` 경로로 서빙 |

> ⚠️ React, Next.js, Supabase 등은 **절대 사용하지 않는다**.

---

## 3. 사용자

- **최고 관리자** (`manager_###` 계정)
  - 관리자 웹의 유일한 사용자
  - 모든 관리 기능에 접근 가능
- 일반 회원은 관리자 웹에 접근할 수 없다

---

## 4. 범위

### In Scope (MVP)
- AW-001: 관리자 로그인
- AW-002: 가입 승인 목록
- AW-003: 회원 목록 (검색/필터)
- AW-004: 회원 관리카드 (상세 + 권한 + Audit Log)
- AW-005: 직책/권한 관리 (권한 매트릭스)
- AW-006: 게시판 관리
- AW-007: 공지사항 관리
- AW-008: 일정 관리
- AW-009: 관리자 계정 보안

### Out of Scope (추후 확장)
- 통계/분석 대시보드
- 배너 광고 관리 (Should-have)
- 참석 투표 관리자 뷰 고급 기능 (Should-have)

---

## 5. 디자인 시스템 (관리자 웹 전용)

### 5-1. 레이아웃

```
┌──────────────────────────────────────────────┐
│                 Header (64px)                 │
├──────────┬───────────────────────────────────┤
│          │                                    │
│ Sidebar  │         Main Content               │
│ (240px)  │         (Padding 32px)             │
│          │                                    │
│          │                                    │
│          │                                    │
│          │                                    │
└──────────┴───────────────────────────────────┘
```

- **좌측 사이드바**: 240px 고정, 네비게이션 메뉴
- **상단 헤더**: 64px 고정, 관리자 이름 + 로그아웃
- **우측 메인 콘텐츠**: Padding 32px

### 5-2. 색상

| 용도 | 색상 | HEX |
|------|------|-----|
| Primary (버튼, 링크, 선택 상태) | Blue | `#2563EB` |
| Danger (삭제, 거절, 경고) | Red | `#DC2626` |
| Text Primary | Dark | `#111827` |
| Text Secondary | Gray | `#6B7280` |
| Border / Divider | Light Gray | `#D1D5DB` |
| Background | White/Light | `#FFFFFF` / `#F9FAFB` |
| Sidebar Background | Dark Gray | `#1F2937` |
| Sidebar Text | White | `#FFFFFF` |

### 5-3. 컴포넌트 규격

| 컴포넌트 | 규격 |
|---------|------|
| 카드 | Border Radius 16px, Shadow sm |
| Content Padding | 32px |
| Modal Width | 480px |
| 버튼 높이 | 40px (기본), 48px (CTA) |
| 버튼 위치 | **우측 정렬** |
| 위험 액션 버튼 | `#DC2626` 배경 또는 테두리 |
| 테이블 행 높이 | 52px |
| 사이드바 메뉴 항목 | 44px |

### 5-4. 앱(모바일 웹)과의 디자인 일관성

이 디자인 톤(그레이톤, Primary Blue, Danger Red)은 **회원용 앱(모바일 웹)에도 일관되게 적용**한다.

| 항목 | 앱 (기존) | 앱 (변경 방향) | 관리자 웹 |
|------|----------|---------------|----------|
| Primary | `#1F4FD8` | `#2563EB` 통일 검토 | `#2563EB` |
| Danger | `#DC2626` | 동일 | `#DC2626` |
| Text Primary | `#111827` | 동일 | `#111827` |
| Text Secondary | `#6B7280` | 동일 | `#6B7280` |

> 디자이너 에이전트에게: 앱 Primary 색상 `#1F4FD8` → `#2563EB` 통일 여부 사장님 확인 필요

---

## 6. 보안 정책

- 관리자 인증 실패 시 모든 화면 접근 차단
- JWT 토큰 만료 시 자동 로그아웃 → 로그인 화면 리다이렉트
- 관리자 계정 비밀번호: 최소 8자, 영문+숫자+특수문자
- 모든 데이터 변경에 **Audit Log** 기록 (누가/언제/무엇을)
- 위험 액션(삭제, 정지, 거절)은 반드시 **확인 모달** 표시
