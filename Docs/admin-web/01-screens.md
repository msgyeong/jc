## AW-001: 관리자 로그인

### 화면 목적
최고 관리자가 admin_id와 password로 인증한다.

### 레이아웃
```
┌────────────────────────────────────┐
│                                    │
│         영등포JC 관리자             │
│                                    │
│    ┌─────────────────────────┐    │
│    │ 관리자 ID               │    │
│    └─────────────────────────┘    │
│    ┌─────────────────────────┐    │
│    │ 비밀번호                 │    │
│    └─────────────────────────┘    │
│                                    │
│    ┌─────────────────────────┐    │
│    │        로그인            │    │  ← Primary Blue
│    └─────────────────────────┘    │
│                                    │
│    ⚠️ 에러 메시지 (인라인)         │
│                                    │
└────────────────────────────────────┘
```

### 기능 상세
- 입력: admin_id (예: manager_008), password
- 인증: `POST /api/admin/auth/login` → JWT 토큰 발급
- 성공: AW-002(가입 승인 목록)으로 이동
- 실패: 인라인 에러 메시지 ("아이디 또는 비밀번호가 일치하지 않습니다")
- 입력값은 실패 시에도 유지 (비밀번호 제외)

### API
- `POST /api/admin/auth/login` → `{ admin_id, password }` → `{ token, admin }`

## AW-002: 가입 승인 목록

### 화면 목적
신규 가입 요청을 승인하거나 거부한다.

### 레이아웃
```
┌─ Header ─────────────────────────────────┐
├─ Sidebar ─┬──────────────────────────────┤
│ 📋 가입승인│  가입 승인 관리               │
│ 👥 회원관리│                              │
│ 🏷️ 직책   │  [대기중 3] [승인됨] [거절됨] │ ← 상태 탭 필터
│ 📰 게시판 │                              │
│ 📢 공지   │  ┌─────────────────────────┐ │
│ 📅 일정   │  │ 이름  이메일  신청일  액션│ │ ← 테이블
│ ──────── │  │ 김OO  a@b   03-09  [승인][거절] │
│ ⚙️ 설정   │  │ 박OO  c@d   03-08  [승인][거절] │
│ 🚪 로그아웃│  └─────────────────────────┘ │
└───────────┴──────────────────────────────┘
```

### 기능 상세
- **상태 탭**: 대기중 / 승인됨 / 거절됨 (기본: 대기중)
- **테이블 컬럼**: 이름, 이메일, 전화번호, 신청일, 상태, 액션
- **승인 버튼**: 클릭 → 확인 모달 → `POST /api/admin/members/:id/approve`
- **거절 버튼**: 클릭 → 거절 사유 입력 모달 → `POST /api/admin/members/:id/reject`
  - 거절 사유는 필수 입력
  - 거절된 회원이 로그인 시 거절 사유가 표시됨
- **대기중 탭 숫자 배지**: 신규 대기 건수 표시
- 승인/거절 처리 후 목록 즉시 갱신

### API
- `GET /api/admin/members/pending` → 대기중 목록
- `GET /api/admin/members?status=approved` → 승인됨 목록
- `GET /api/admin/members?status=rejected` → 거절됨 목록
- `POST /api/admin/members/:id/approve`
- `POST /api/admin/members/:id/reject` → `{ rejection_reason }`

## AW-003: 회원 목록

### 화면 목적
전체 회원을 검색/필터링하여 조회하고, 회원 관리카드(AW-004)로 진입한다.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────┐
│            │  회원 목록              [Excel↓]  │
│            │                                  │
│            │  [검색: ________] [상태▼] [기수▼] │ ← 필터
│            │                                  │
│            │  ┌────────────────────────────┐  │
│            │  │ 이름  직책   소속  기수 상태│  │
│            │  │ 김OO  회원   40기  정상  →  │  │ ← 클릭 → AW-004
│            │  │ 박OO  이사   39기  정상  →  │  │
│            │  │ 이OO  (없음) 41기  대기  →  │  │
│            │  └────────────────────────────┘  │
│            │                                  │
│            │  < 1 2 3 ... 5 >                 │ ← 페이지네이션
└────────────┴──────────────────────────────────┘
```

### 기능 상세
- **검색**: 이름, 이메일, 전화번호로 통합 검색
- **필터**: 상태(전체/정상/대기/정지/탈퇴), 기수, 소속, 직책
- **테이블 컬럼**: 이름, 직책, 소속, 기수, 상태, 가입일
- **행 클릭**: AW-004 회원 관리카드로 이동
- **페이지네이션**: 50명 단위
- **Excel 다운로드** (Should-have): 현재 필터 결과를 CSV/Excel로 내보내기

### API
- `GET /api/admin/members?search=&status=&cohort=&page=&limit=50`

## AW-004: 회원 관리카드

### 화면 목적
회원의 전체 정보를 조회/수정하고, 권한을 설정하며, 변경 이력을 확인한다.
입회원서 디지털화(R-01)의 핵심 화면.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────────────┐
│            │  ← 회원 목록    회원 관리카드    [인쇄]    │
│            │                                          │
│            │  ┌─ 기본 정보 ──────────────────────┐    │
│            │  │ 이름: 김OO        상태: [정상 ▼]  │    │
│            │  │ 이메일: a@b.com   전화: 010-****  │    │
│            │  │ 가입일: 2026-01-15               │    │
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │  ┌─ JC 정보 (관리자 전용) ──────────┐    │
│            │  │ 직책: [회원 ▼]    소속: [영등포JC]│    │  ← 관리자만 수정
│            │  │ 기수: [41기]      가입 연도: 2026 │    │
│            │  │ 교육 이력: [수정]                 │    │
│            │  │ JC 경력: [수정]                   │    │
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │  ┌─ 개인 정보 ──────────────────────┐    │
│            │  │ 직장: OO회사 / 대리               │    │
│            │  │ 업종: IT                          │    │
│            │  │ 주소: 서울시 영등포구...           │    │
│            │  │ 가족: (JSONB 데이터)              │    │
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │  ┌─ 권한 설정 ──────────────────────┐    │
│            │  │ ☑ 공지 작성  ☑ 일정 등록          │    │
│            │  │ ☐ 게시글 삭제                     │    │
│            │  │             [권한 저장]            │    │ ← 우측 정렬
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │  ┌─ 변경 이력 (Audit Log) ──────────┐    │
│            │  │ 2026-03-09 직책 변경: 회원→이사   │    │
│            │  │   변경자: manager_008              │    │
│            │  │ 2026-01-15 가입 승인               │    │
│            │  │   변경자: manager_008              │    │
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │           [회원 정지]  [저장]             │ ← 정지: 빨간, 저장: 파란
└────────────┴──────────────────────────────────────────┘
```

### 기능 상세

#### 기본 정보 섹션
- 이름, 이메일, 전화번호, 가입일, 상태
- 상태 변경 드롭다운: 정상/정지/탈퇴 처리 (확인 모달 필수)

#### JC 정보 섹션 (관리자 전용 필드)
- **직책**: 드롭다운 선택 (회장/상임부회장/내무부회장/사무국장/사무차장/이사/회원 등)
  - 변경 시 **권한 자동 반영** (R-03)
  - 변경 이력 자동 기록
- **소속**: 영등포JC (Phase 2에서 멀티 테넌트 시 확장)
- **기수**: 신입회원교육 기수
- **교육 이력**: 팝업/모달로 교육 목록 관리 (교육명, 이수 여부, 이수일)
- **JC 경력**: 팝업/모달로 경력 목록 관리 (직책, 연도, 비고)
- **표창/국제대회**: 관리자 입력

#### 개인 정보 섹션
- 회원이 가입 시 입력한 정보 조회 (읽기 전용 또는 관리자 보완 입력)
- 직장, 업종, 주소, 가족 정보 등

#### 권한 설정 섹션
- 체크박스: 공지 작성 / 일정 등록 / 게시글 삭제
- 직책 변경 시 자동으로 체크 상태 업데이트 (직책-권한 매트릭스 기반)
- 수동 오버라이드 가능 (특정 회원에게만 추가 권한 부여)

#### 변경 이력 (Audit Log)
- 시간순 역순으로 모든 변경 기록 표시
- 항목: 변경 일시, 변경 내용, 변경자
- 대상: 상태 변경, 직책 변경, 권한 변경, 정보 수정

#### 인쇄 기능
- "인쇄" 버튼 → 회원관리카드를 인쇄용 레이아웃으로 표시 (CSS @media print)
- 입회원서 디지털화(R-01) 대응

### API
- `GET /api/admin/members/:id` → 전체 회원 정보
- `PUT /api/admin/members/:id` → 정보 수정
- `PUT /api/admin/members/:id/position` → 직책 변경 (+ 권한 자동 반영)
- `PUT /api/admin/members/:id/permissions` → 권한 설정
- `PUT /api/admin/members/:id/status` → 상태 변경 (정지/복구/탈퇴)
- `GET /api/admin/members/:id/audit-log` → 변경 이력 조회
- `PUT /api/admin/members/:id/education` → 교육 이력 수정
- `PUT /api/admin/members/:id/career` → JC 경력 수정

## AW-005: 직책/권한 관리

### 화면 목적
직책별 권한 매트릭스를 관리한다. 여기서 설정한 권한이 앱에 실시간 반영된다.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────────────┐
│            │  직책/권한 관리                           │
│            │                                          │
│            │  ┌─ 권한 매트릭스 ───────────────────┐    │
│            │  │            │ 공지작성│일정등록│삭제 │    │
│            │  │ ───────────┼────────┼───────┼──── │    │
│            │  │ 회장       │   ✅   │  ✅   │ ✅  │    │
│            │  │ 상임부회장  │   ✅   │  ✅   │ ✅  │    │
│            │  │ 내무부회장  │   ✅   │  ✅   │ ✅  │    │
│            │  │ 사무국장    │   ✅   │  ✅   │ ✅  │    │
│            │  │ 사무차장    │   ✅   │  ✅   │ ☐  │    │
│            │  │ 이사       │   ☐   │  ☐   │ ☐  │    │
│            │  │ 회원       │   ☐   │  ☐   │ ☐  │    │
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │  [+ 직책 추가]              [저장]        │
│            │                                          │
│            │  ⚠️ 저장 시 해당 직책의 모든 회원에게       │
│            │     즉시 반영됩니다.                       │
└────────────┴──────────────────────────────────────────┘
```

### 기능 상세
- **권한 매트릭스**: 직책(행) × 권한(열) 체크박스 테이블
- **권한 종류**: 공지사항 작성, 일정 등록, 게시글/댓글 삭제
- **직책 추가**: 새 직책명 입력 → 행 추가
- **저장**: 변경된 매트릭스를 일괄 저장
  - 저장 시 해당 직책을 가진 **모든 회원의 권한이 즉시 반영**
  - 확인 모달: "변경 사항이 즉시 반영됩니다. 저장하시겠습니까?"
- **하드코딩 금지**: 직책/권한은 모두 DB 테이블에서 관리

### API
- `GET /api/admin/positions` → 직책 목록 + 권한 매트릭스
- `PUT /api/admin/positions` → 매트릭스 일괄 저장
- `POST /api/admin/positions` → 직책 추가

## AW-006: 게시판 관리

### 화면 목적
게시글 목록을 조회하고, 상세 내용을 Drawer로 확인하며, 필요 시 삭제한다.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────────────┐
│            │  게시판 관리                              │
│            │                                          │
│            │  [공지 게시판] [일반 게시판] ← 탭          │
│            │  [검색: ________]                        │
│            │                                          │
│            │  ┌────────────────────────────────────┐  │
│            │  │ 제목      작성자  작성일  조회 댓글  │  │
│            │  │ 3월 회의..  김OO  03-09  25  3    │  │ ← 클릭 → Drawer
│            │  │ 봄 행사..  박OO  03-08  18  1    │  │
│            │  └────────────────────────────────────┘  │
│            │                                          │
│            │  ┌─ Drawer (우측 슬라이드) ──────────┐    │
│            │  │ 제목: 3월 회의 결과 공유           │    │
│            │  │ 작성자: 김OO | 2026-03-09         │    │
│            │  │ ─────────────────────────────     │    │
│            │  │ 본문 내용...                      │    │
│            │  │ [이미지 1] [이미지 2]              │    │
│            │  │ ─────────────────────────────     │    │
│            │  │ 댓글 (3)                          │    │
│            │  │  ├ 좋은 글이네요 — 박OO  [삭제]    │    │
│            │  │  └ 감사합니다 — 이OO    [삭제]     │    │
│            │  │ ─────────────────────────────     │    │
│            │  │           [게시글 삭제]            │    │  ← Danger Red
│            │  └──────────────────────────────────┘    │
└────────────┴──────────────────────────────────────────┘
```

### 기능 상세
- **탭**: 공지 게시판 / 일반 게시판
- **검색**: 제목, 작성자로 검색
- **테이블 컬럼**: 제목, 작성자, 작성일, 조회수, 댓글수, 고정 여부
- **행 클릭 → Drawer**: 우측에서 슬라이드 인, 게시글 상세 표시
  - 본문, 이미지, 댓글 목록
  - 개별 댓글 삭제 버튼
  - 게시글 삭제 버튼 (Danger Red, 확인 모달)
- **고정 토글** (공지 게시판만): 고정/해제
- 삭제는 soft delete

### API
- `GET /api/admin/posts?category=notice&search=&page=`
- `DELETE /api/admin/posts/:id` (확인 후)
- `DELETE /api/admin/posts/:id/comments/:commentId`
- `POST /api/admin/posts/:id/pin` (토글)

## AW-007: 공지사항 관리

### 화면 목적
공지사항을 작성/수정/삭제하고, 고정 기능을 관리한다.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────────────┐
│            │  공지사항 관리              [+ 공지 작성]  │
│            │                                          │
│            │  ┌────────────────────────────────────┐  │
│            │  │ 📌 제목       작성일  조회 고정 액션│  │
│            │  │ 📌 연회비 안내  03-05  120  ✅  [수정][삭제]│
│            │  │    3월 회의    03-09   25  ☐  [수정][삭제]│
│            │  └────────────────────────────────────┘  │
│            │                                          │
│            │  ┌─ 작성/수정 모달 (480px) ──────────┐    │
│            │  │ 제목: [________________]           │    │
│            │  │ 본문: [                           ]│    │
│            │  │       [                           ]│    │
│            │  │ 이미지: [+ 첨부] (최대 5개)        │    │
│            │  │ ☐ 고정 공지로 설정                 │    │
│            │  │ ☐ 참석 투표 활성화                 │    │
│            │  │   └ 투표 유형: [이사회 ▼]          │    │
│            │  │   └ 마감일: [2026-03-20]           │    │
│            │  │ ☐ 일정 연동                        │    │
│            │  │   └ 일정일: [2026-03-25]           │    │
│            │  │   └ 시간: [14:00]                  │    │
│            │  │   └ 장소: [________________]       │    │
│            │  │                                    │    │
│            │  │              [취소]  [저장]         │    │
│            │  └──────────────────────────────────┘    │
└────────────┴──────────────────────────────────────────┘
```

### 기능 상세
- **공지 목록**: 고정 공지 상단, 최신순 정렬
- **작성 모달**: 제목, 본문, 이미지(최대 5개), 고정 여부
  - **참석 투표 활성화** (R-02): 체크 시 투표 유형/마감일 입력
  - **일정 연동** (R-07): 체크 시 일정 날짜/시간/장소 입력 → 캘린더 자동 반영
- **수정**: 동일 모달에서 기존 내용 로드 후 수정
- **삭제**: 확인 모달 → soft delete
- **고정 토글**: 행에서 바로 토글

### API
- `GET /api/admin/notices`
- `POST /api/admin/notices` → `{ title, content, images, is_pinned, attendance_config, schedule_link }`
- `PUT /api/admin/notices/:id`
- `DELETE /api/admin/notices/:id`

## AW-008: 일정 관리

### 화면 목적
일정을 등록/수정/삭제한다.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────────────┐
│            │  일정 관리                  [+ 일정 등록]  │
│            │                                          │
│            │  [2026년 3월 ▼]  [전체▼] ← 월/카테고리   │
│            │                                          │
│            │  ┌────────────────────────────────────┐  │
│            │  │ 날짜     제목       장소   카테고리  │  │
│            │  │ 03/15   정기 총회    시청  🔴 행사   │  │
│            │  │ 03/20   신입 교육    OO빌딩 🟡 교육  │  │
│            │  │ 03/25   이사회      사무실 🔵 회의   │  │
│            │  └────────────────────────────────────┘  │
│            │                                          │
│            │  ┌─ 등록/수정 모달 (480px) ──────────┐    │
│            │  │ 제목: [________________]           │    │
│            │  │ 날짜: [2026-03-25]                 │    │
│            │  │ 시간: [14:00] ~ [16:00]            │    │
│            │  │ 장소: [________________]           │    │
│            │  │ 카테고리: [행사 ▼]                  │    │
│            │  │ 설명: [                           ]│    │
│            │  │                                    │    │
│            │  │    [삭제]          [취소]  [저장]   │    │
│            │  └──────────────────────────────────┘    │
└────────────┴──────────────────────────────────────────┘
```

### 기능 상세
- **필터**: 월 선택, 카테고리 선택 (행사/회의/교육/공휴일/기타)
- **테이블 컬럼**: 날짜, 제목, 장소, 카테고리(컬러 도트), 연결 공지
- **등록 모달**: 제목, 날짜, 시간(시작/종료), 장소, 카테고리, 설명
- **수정**: 행 클릭 → 모달에 기존 데이터 로드
- **삭제**: 모달 내 삭제 버튼 (Danger Red, 확인 모달)
- **연결된 공지 표시**: 공지-일정 연동(R-07)으로 생성된 일정은 연결 공지 링크 표시

### API
- `GET /api/admin/schedules?month=2026-03&category=`
- `POST /api/admin/schedules`
- `PUT /api/admin/schedules/:id`
- `DELETE /api/admin/schedules/:id`

## AW-009: 관리자 계정 보안

### 화면 목적
관리자 계정의 비밀번호를 변경하고 보안 정책을 관리한다.

### 레이아웃
```
┌─ Sidebar ─┬──────────────────────────────────────────┐
│            │  계정 설정                                │
│            │                                          │
│            │  ┌─ 비밀번호 변경 ───────────────────┐    │
│            │  │ 현재 비밀번호: [________________]  │    │
│            │  │ 새 비밀번호:   [________________]  │    │
│            │  │ 비밀번호 확인: [________________]  │    │
│            │  │                                   │    │
│            │  │ · 최소 8자 이상                    │    │
│            │  │ · 영문 + 숫자 + 특수문자 포함       │    │
│            │  │                                   │    │
│            │  │                     [비밀번호 변경] │    │
│            │  └──────────────────────────────────┘    │
│            │                                          │
│            │  ┌─ 계정 정보 ──────────────────────┐    │
│            │  │ 관리자 ID: manager_008             │    │
│            │  │ 마지막 로그인: 2026-03-09 14:30    │    │
│            │  └──────────────────────────────────┘    │
└────────────┴──────────────────────────────────────────┘
```

### 기능 상세
- **비밀번호 변경**: 현재 비밀번호 확인 → 새 비밀번호 입력 → 확인
- **비밀번호 정책**: 최소 8자, 영문+숫자+특수문자
- 변경 성공 시 토큰 재발급 (세션 유지)
- **계정 정보**: 관리자 ID, 마지막 로그인 시각 표시

### API
- `PUT /api/admin/auth/password` → `{ current_password, new_password }`
- `GET /api/admin/auth/me` → 계정 정보
