본 문서는 **관리자 웹**의 화면별 텍스트 와이어프레임을 정의한다.

- 화면 목록: `./01-screens.md`
- 설계 기준: `../prd/08-design.md`

---

## [AW-001 관리자 웹 로그인]

### 화면 목적
- 최고 관리자가 관리자 웹에 로그인한다.

### UI 구성 요소
- 아이디 입력
- 비밀번호 입력
- 로그인 버튼

### 동작
- 인증 실패 시 인라인 오류 메시지 표시
- 입력값은 유지한다.

---

## [AW-002 가입 승인 목록]

### 화면 목적
- 가입 대기 회원을 승인/거절한다.

### UI 구성 요소
- 가입 대기 리스트
- 필터(상태)
- 승인/거절 액션
- 거절 사유 입력(거절 시)

### 동작
- 승인/거절 처리 결과는 즉시 목록에 반영한다.
- 거절 시 거절 사유는 회원에게 노출될 수 있으므로(앱 로그인 차단 시),
  작성 톤을 관리한다.

---

## [AW-003 회원 목록]

### 화면 목적
- 전체 회원을 조회/검색한다.
- 회원 상세로 진입한다.

### UI 구성 요소
- 검색 입력(이름/기수/소속 등 PRD 정의 범위 내)
- 필터(상태: 가입대기/승인/정지/탈퇴 등)
- 회원 리스트(핵심 식별 정보 요약)

### 동작
- 회원 선택 시 AW-004로 이동한다.

---

## [AW-004 회원 상세]

### 화면 목적
- 회원 정보를 조회/수정하고, 직책/권한을 관리한다.

### UI 구성 요소
- 기본 정보 섹션
- 관리자 전용 필드 섹션(읽기/수정 정책은 PRD 기준)
- 권한/직책 설정 섹션

### 동작
- 변경 사항 저장 시 앱에 즉시 반영한다.
- 변경 이력(누가/언제/무엇을)을 남길 수 있도록 구성(권장)

---

## [AW-005 직책/권한 관리]

### 화면 목적
- 직책별 권한을 부여·회수한다.
- 앱 내 관리자(직책 기반) 권한을 운영한다.

### UI 구성 요소
- 직책 목록
- 직책별 권한 체크(공지사항 작성, 일정 등록 등)
- 저장 버튼

### 동작
- 저장 시 권한은 즉시 반영된다.

---

## [AW-006 게시판 관리]

### 화면 목적
- 게시글 목록을 조회하고 운영 정책에 따라 삭제한다.
- 댓글 삭제 등 운영 액션을 수행한다.

### UI 구성 요소
- 게시글 목록(검색/필터)
- 게시글 상세(운영용)
- 삭제 버튼(확인 다이얼로그 포함)
- 댓글 목록 및 댓글 삭제

### 동작
- 삭제는 되돌릴 수 없으므로 확인 단계를 둔다.

---

## [AW-007 공지사항 관리]

### 화면 목적
- 공지사항 작성/수정/삭제를 수행한다.

### UI 구성 요소
- 공지 목록
- 공지 작성/수정 폼
- 고정 공지 설정(PRD 정의 시)
- 참석자 조사 활성화 옵션(PRD 정의 시)

### 동작
- 작성/수정/삭제 결과는 앱 홈/목록에 즉시 반영된다.

---

## [AW-008 일정 관리]

### 화면 목적
- 일정을 등록/수정/삭제하고 운영한다.

### UI 구성 요소
- 일정 목록(검색/필터)
- 일정 등록/수정 폼(제목/날짜/시간/장소/설명)
- 삭제 버튼(확인 다이얼로그 포함)

### 동작
- 등록/수정/삭제 결과는 앱 홈/목록에 즉시 반영된다.

---

## [AW-009 관리자 계정 보안]

### 화면 목적
- 관리자 계정 보안을 생성/유지/관리한다.

### UI 구성 요소
- 비밀번호 생성/변경 UI
- 보안 정책 안내(최소 길이, 재사용 제한 등: 정책은 추후 확정)

### 동작
- 변경 후 재로그인이 필요한지 정책으로 명확히 정의한다(추후 확정).
