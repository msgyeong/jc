## UF-AW-001: 관리자 로그인

```
[AW-001 로그인 화면]
    ↓ admin_id + password 입력
    ↓ POST /api/admin/auth/login
    ├── 성공 → JWT 토큰 저장 → [AW-002 가입 승인 목록]
    └── 실패 → 인라인 에러 메시지 (비밀번호 초기화, 입력값 유지)
```

---

## UF-AW-002: 가입 승인 처리

```
[AW-002 가입 승인 목록]
    ↓ 대기중 탭 선택 (기본)
    ↓ 회원 선택
    ├── [승인] 클릭
    │   ↓ 확인 모달: "승인하시겠습니까?"
    │   ├── 확인 → POST /approve → 목록 갱신 → 성공 토스트
    │   └── 취소 → 모달 닫기
    └── [거절] 클릭
        ↓ 거절 사유 입력 모달
        ↓ 사유 입력 (필수)
        ├── 확인 → POST /reject → 목록 갱신 → 성공 토스트
        └── 취소 → 모달 닫기
```

## UF-AW-003: 회원 조회 → 관리카드

```
[AW-003 회원 목록]
    ↓ 검색/필터 조건 입력
    ↓ 결과 테이블 표시
    ↓ 회원 행 클릭
    ↓
[AW-004 회원 관리카드]
    ├── 기본 정보 확인/수정
    ├── JC 정보 수정 (관리자 전용)
    │   ├── 직책 변경 → 권한 자동 반영 + Audit Log
    │   ├── 교육 이력 수정 (모달)
    │   └── JC 경력 수정 (모달)
    ├── 권한 설정 (체크박스)
    │   └── [권한 저장] → PUT /permissions
    ├── 상태 변경
    │   ├── [정지] → 확인 모달 → PUT /status
    │   └── [탈퇴] → 확인 모달 → PUT /status
    ├── [인쇄] → 인쇄용 레이아웃 → 브라우저 인쇄
    └── [저장] → PUT /members/:id → 성공 토스트
```

## UF-AW-004: 직책/권한 매트릭스 관리

```
[AW-005 직책/권한 관리]
    ↓ 현재 매트릭스 로드 (GET /positions)
    ↓ 체크박스 변경 (직책별 권한 토글)
    ├── [+ 직책 추가] → 직책명 입력 → 행 추가
    └── [저장] 클릭
        ↓ 확인 모달: "변경 사항이 즉시 반영됩니다"
        ├── 확인 → PUT /positions → 전체 앱 즉시 반영
        └── 취소 → 모달 닫기
```

## UF-AW-005: 게시글 관리 (Drawer)

```
[AW-006 게시판 관리]
    ↓ 공지/일반 탭 선택
    ↓ 게시글 행 클릭
    ↓
[Drawer 오픈 (우측 슬라이드)]
    ├── 게시글 상세 (본문, 이미지)
    ├── 댓글 목록
    │   └── 개별 댓글 [삭제] → 확인 모달 → DELETE /comments/:id
    ├── [고정/해제] (공지만) → POST /pin
    └── [게시글 삭제] (Danger Red)
        ↓ 확인 모달: "정말 삭제하시겠습니까?"
        ├── 확인 → DELETE /posts/:id → Drawer 닫기 + 목록 갱신
        └── 취소 → 모달 닫기
```

## UF-AW-006: 공지사항 작성 (참석 투표 + 일정 연동)

```
[AW-007 공지사항 관리]
    ↓ [+ 공지 작성] 클릭
    ↓
[작성 모달 (480px)]
    ↓ 제목, 본문, 이미지 입력
    ├── ☐ 고정 공지 설정 (선택)
    ├── ☐ 참석 투표 활성화 (R-02)
    │   ↓ 체크 시 추가 필드 표시
    │   ├── 투표 유형: [이사회/월례회/기타]
    │   └── 마감일: [날짜 선택]
    ├── ☐ 일정 연동 (R-07)
    │   ↓ 체크 시 추가 필드 표시
    │   ├── 일정 날짜: [날짜 선택]
    │   ├── 시간: [시간 입력]
    │   └── 장소: [텍스트 입력]
    └── [저장] 클릭
        ↓ POST /notices (with attendance_config + schedule_link)
        ↓ 일정 연동 체크 시 → 일정 테이블에 자동 생성
        ↓ 모달 닫기 + 목록 갱신
```

## UF-AW-007: 일정 등록/수정/삭제

```
[AW-008 일정 관리]
    ├── [+ 일정 등록] 클릭
    │   ↓ 등록 모달 (480px)
    │   ↓ 제목, 날짜, 시간, 장소, 카테고리, 설명 입력
    │   └── [저장] → POST /schedules → 모달 닫기 + 목록 갱신
    │
    ├── 일정 행 클릭 (수정)
    │   ↓ 수정 모달 (기존 데이터 로드)
    │   ├── 정보 수정 → [저장] → PUT /schedules/:id
    │   └── [삭제] (Danger Red)
    │       ↓ 확인 모달
    │       ├── 확인 → DELETE /schedules/:id
    │       └── 취소 → 모달 닫기
    │
    └── 연결 공지 링크 클릭
        ↓ AW-007 공지사항 관리로 이동 (해당 공지 포커스)
```

## UF-AW-008: 비밀번호 변경

```
[AW-009 계정 설정]
    ↓ 현재 비밀번호 입력
    ↓ 새 비밀번호 입력
    ↓ 비밀번호 확인 입력
    ↓ [비밀번호 변경] 클릭
    ├── 유효성 검사 통과
    │   ↓ PUT /auth/password
    │   ├── 성공 → 토큰 재발급 + 성공 메시지
    │   └── 실패 (현재 비밀번호 불일치) → 인라인 에러
    └── 유효성 검사 실패 (정책 미충족) → 인라인 에러
```
