## 1. 요청 개요

Web Push 알림 기능 추가에 따라 아래 UI 컴포넌트의 디자인 명세가 필요합니다.

---

## 2. 필요한 UI 컴포넌트

### 2-1. 앱바 알림 아이콘

- **위치**: 앱바 우측 (기존 위치 유지, 아이콘 추가)
- **아이콘**: 종 모양 (🔔)
- **미읽음 배지**: 숫자 표시 (빨간 원, danger 색상 `#DC2626`)
  - 0이면 배지 숨김
  - 99 이상이면 "99+" 표시
- **배지 크기/위치**: 아이콘 우상단, pill 형태

```
참고 레이아웃:
┌──────────────────────────────────────┐
│  영등포 JC              🔔          │
│                          ③          │  ← 빨간 배지
└──────────────────────────────────────┘
```

**디자이너 결정 필요**:
- 배지 크기 (16px? 18px?)
- 배지 위치 오프셋
- 숫자 폰트 크기
- 아이콘 크기 (24px?)

---

### 2-2. 알림 센터 화면

- **전체 레이아웃**: 일반 SPA 화면 (탭 형식 아님, 앱바 + 리스트)
- **앱바**: "← 알림" + [모두 읽음 처리] 텍스트 버튼

#### 알림 리스트 아이템

각 알림 항목의 구성:

```
┌─────────────────────────────────────────┐
│  ● [아이콘] 알림 제목                    │
│     알림 본문 (최대 2줄, 말줄임)          │
│     10분 전                             │
└─────────────────────────────────────────┘
```

- **미읽음 표시**: 좌측 파란 점 (● primary 색상 `#2563EB`, 8px)
- **읽음 상태**: 파란 점 없음, 텍스트 색상 `text-sub` (#6B7280)
- **유형 아이콘**: 📢 (공지) / 📅 (일정) / ⏰ (리마인더) / 💬 (댓글)
- **시간 표시**: `text-hint` 색상 (#9CA3AF), caption 크기

**디자이너 결정 필요**:
- 아이템 높이 / 패딩
- 아이콘 크기 및 배경색 (원형 배경? 없음?)
- 미읽음/읽음 항목의 배경색 차이 (미읽음: `#EFF6FF`? 읽음: `#FFFFFF`?)
- 구분선 스타일 (border? gap?)
- 스와이프 삭제 지원 여부

#### 빈 상태

```
┌─────────────────────────────────────┐
│                                     │
│          🔔                         │
│     알림이 없습니다                  │
│                                     │
└─────────────────────────────────────┘
```

---

### 2-3. 알림 설정 화면

- **진입**: 프로필 > 알림 설정
- **레이아웃**: 설정 목록 (토글 스위치)

```
┌─────────────────────────────────────┐
│  ← 알림 설정                        │
├─────────────────────────────────────┤
│                                     │
│  🔔 전체 알림                [ON]   │  ← 마스터 스위치
│                                     │
│  ─── 알림 유형 ─────────────────── │
│                                     │
│  📢 공지 알림                [ON]   │
│     새 공지사항이 등록되면 알림      │
│                                     │
│  📅 일정 알림                [ON]   │
│     일정 등록/변경/취소 시 알림      │
│                                     │
│  ⏰ 일정 리마인더            [ON]   │
│     일정 하루 전 오전 9시 알림       │
│                                     │
│  💬 댓글 알림                [ON]   │
│     내 글에 댓글이 달리면 알림       │
│                                     │
└─────────────────────────────────────┘
```

**디자이너 결정 필요**:
- 토글 스위치 디자인 (크기, 색상: ON=primary, OFF=neutral)
- 마스터 스위치와 하위 토글의 시각적 구분 (크기 차이? 배경? 구분선?)
- 마스터 OFF 시 하위 토글 비활성 스타일
- 각 항목의 설명 텍스트 스타일

---

### 2-4. 푸시 구독 유도 배너

로그인 후, 푸시 미구독 상태에서 노출되는 배너:

```
┌─────────────────────────────────────┐
│  🔔 알림을 켜면 공지와 일정을        │
│     바로 받아볼 수 있어요!           │
│                                     │
│  [알림 켜기]              [닫기]    │
└─────────────────────────────────────┘
```

**디자이너 결정 필요**:
- 배너 위치 (상단 고정? 하단 플로팅? 카드 형태?)
- 배경색 (`primary-bg` #EFF6FF? 또는 `primary-surface` #E6ECFA?)
- [알림 켜기] 버튼 스타일 (Primary? Ghost?)
- [닫기] 버튼 스타일 (텍스트? X 아이콘?)
- 애니메이션 (slide-down? fade-in?)

---

### 2-5. 홈 화면 추가 안내 배너

PWA가 아닌 브라우저 접속 시 노출:

```
┌─────────────────────────────────────┐
│  📱 홈 화면에 추가하면              │
│     알림을 받을 수 있어요!           │
│                                     │
│  [추가 방법 보기]      [닫기]       │
└─────────────────────────────────────┘
```

**디자이너 결정 필요**:
- 구독 유도 배너와 동일한 스타일로 통일할지
- OS별 안내 모달 디자인 (Android/iOS 각각 스크린샷 포함 여부)

---

### 2-6. 푸시 알림 디자인 (OS 알림 영역)

OS 알림 센터에 표시되는 푸시 알림의 내용 구성:

```
┌─────────────────────────────────────┐
│  [앱 아이콘] 영등포 JC    · 방금     │
│  📢 새 공지사항                      │
│  3월 정기 이사회 안내 — 3월 정기     │
│  이사회를 안내합니다...              │
└─────────────────────────────────────┘
```

**디자이너 결정 필요**:
- 앱 아이콘 디자인 (192x192, 512x512)
- 배지 아이콘 (72x72, 모노크롬 — Android 상태바용)
- maskable 아이콘 (safe zone 내 배치)

## 3. 디자인 토큰 참조

기존 `07-unified-design-guide.md` 기준:

| 용도 | 토큰 | 값 |
|------|------|-----|
| 미읽음 점 | primary | `#2563EB` |
| 미읽음 배경 | primary-bg | `#EFF6FF` |
| 배지 배경 | danger | `#DC2626` |
| 배지 텍스트 | white | `#FFFFFF` |
| 읽음 텍스트 | text-sub | `#6B7280` |
| 시간 텍스트 | text-hint | `#9CA3AF` |
| 토글 ON | primary | `#2563EB` |
| 토글 OFF | border | `#D1D5DB` |
| 배너 배경 | primary-bg | `#EFF6FF` |

## 4. 아이콘 제작 요청

PWA 아이콘 세트 제작 필요:

| 파일명 | 크기 | 용도 |
|--------|------|------|
| `icon-192.png` | 192×192 | PWA 아이콘 |
| `icon-512.png` | 512×512 | PWA 스플래시 |
| `icon-maskable-192.png` | 192×192 (safe zone 80%) | Android adaptive icon |
| `icon-maskable-512.png` | 512×512 (safe zone 80%) | Android adaptive icon |
| `badge-72.png` | 72×72 (모노크롬) | Android 알림 배지 |
| `apple-touch-icon.png` | 180×180 | iOS 홈 화면 아이콘 |

- 테마: Primary Blue `#2563EB` 배경 + 흰색 "JC" 텍스트 (또는 로고)
- maskable 아이콘: 중앙 80% 영역에 로고 배치

## 5. 우선순위

| 순위 | 컴포넌트 | 이유 |
|------|----------|------|
| 1 | PWA 아이콘 세트 | manifest.json 필수 |
| 2 | 푸시 구독 유도 배너 | 최초 구독 전환율 핵심 |
| 3 | 앱바 알림 아이콘 + 배지 | 모든 화면에 노출 |
| 4 | 알림 센터 화면 | Phase 2 구현 |
| 5 | 알림 설정 화면 | Phase 2 구현 |
| 6 | 홈 화면 추가 안내 배너 | Phase 2 구현 |
