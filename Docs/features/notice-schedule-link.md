## 1. 기능 개요

공지사항에 일정 정보를 첨부하면 캘린더에 자동으로 일정이 생성되고, 양방향으로 연결되는 기능.

- **목적**: 공지에 포함된 행사/회의 일정을 수동으로 중복 등록하는 번거로움 제거
- **핵심 원칙**: 공지가 일정의 원본 → 일정은 공지의 파생 데이터
- **현재 상태**: 공지와 일정이 완전히 분리된 독립 기능. `posts` 테이블에 `schedule_id` 예비 컬럼 존재.

## 2. 사용자 흐름

### 2-1. 공지 작성 시 일정 첨부

```
공지 작성 화면
    ↓
[일정 첨부] 토글 ON
    ↓
일정 정보 입력 필드 표시:
  - 일정 제목 (기본값: 공지 제목)
  - 날짜/시간 (시작 ~ 종료)
  - 장소
  - 카테고리 (회의/행사/교육/기타)
    ↓
공지 등록 → 공지 저장 + 일정 자동 생성 (단일 트랜잭션)
    ↓
캘린더에 해당 일정 표시 + 일정에 "연결된 공지" 링크 표시
```

### 2-2. 캘린더에서 연결 공지 보기

```
캘린더 → 일정 상세 진입
    ↓
[🔗 연결된 공지] 링크 표시
    ↓
클릭 → 해당 공지 상세 화면으로 이동
```

### 2-3. 공지 수정 시 연결 일정 동기화

```
공지 수정 → 첨부된 일정 정보도 수정
    ↓
저장 → 연결된 일정도 자동 업데이트 (트랜잭션)
```

## 4. 연동 규칙

### 4-1. 생성 규칙

| 항목 | 규칙 |
|------|------|
| **일정 제목** | 기본값 = 공지 제목. 사용자가 변경 가능 |
| **일정 카테고리** | 사용자 선택 (기본: `event`) |
| **일정 작성자** | 공지 작성자와 동일 (`created_by`) |
| **트랜잭션** | 공지 + 일정이 하나의 트랜잭션으로 저장 (하나라도 실패 시 전체 롤백) |
| **연결 저장** | `posts.linked_schedule_id`, `schedules.linked_post_id` 양방향 참조 |

### 4-2. 수정 규칙

| 수정 대상 | 동기화 범위 |
|-----------|-------------|
| **공지 수정** (일정 첨부 정보 변경) | 연결된 일정도 자동 업데이트 |
| **일정 직접 수정** (캘린더에서) | 공지의 일정 섹션에도 반영 (양방향 동기화) |
| **공지에서 일정 첨부 해제** | 연결된 일정 삭제 여부 확인 다이얼로그 표시 |

### 4-3. 삭제 규칙

| 삭제 대상 | 동작 |
|-----------|------|
| **공지 삭제** | 확인: "연결된 일정도 함께 삭제하시겠습니까?" → 예: 일정도 삭제 / 아니오: 일정만 연결 해제하여 독립 유지 |
| **일정 삭제** (캘린더에서) | 공지의 `linked_schedule_id`를 NULL로 변경. 공지는 유지됨 |

## 5. API 설계안

### 5-1. 공지 작성 시 일정 자동 생성

기존 공지 작성 API를 확장하여 `schedule` 객체 포함 시 일정을 함께 생성.

```
POST /api/posts
Headers: Authorization: Bearer <token>
Body: {
  "title": "3월 정기 이사회 안내",
  "content": "3월 정기 이사회를 안내합니다...",
  "category": "notice",
  "is_pinned": false,
  "schedule": {                           // ← 신규 필드 (선택)
    "title": "3월 정기 이사회",
    "category": "meeting",
    "start_date": "2026-03-15T14:00:00+09:00",
    "end_date": "2026-03-15T16:00:00+09:00",
    "location": "영등포구청 3층 대회의실"
  },
  "vote_config": {                        // ← 투표 설정 (선택)
    "vote_type": "board_meeting",
    "deadline": "2026-03-14T18:00:00+09:00",
    "quorum_count": 20,
    "quorum_basis": "directors"
  }
}
Response: {
  "success": true,
  "data": {
    "post": { "id": 15, "title": "3월 정기 이사회 안내", "linked_schedule_id": 10, ... },
    "schedule": { "id": 10, "title": "3월 정기 이사회", "linked_post_id": 15, ... }
  }
}
```

### 5-2. 공지 수정 시 연결 일정 동기화

```
PUT /api/posts/:id
Body: {
  "title": "3월 정기 이사회 안내 (수정)",
  "content": "...",
  "schedule": {
    "title": "3월 정기 이사회 (날짜 변경)",
    "start_date": "2026-03-22T14:00:00+09:00",
    "end_date": "2026-03-22T16:00:00+09:00",
    "location": "영등포구청 3층 대회의실"
  }
}
```

- 기존 연결 일정이 있으면 UPDATE
- `schedule: null`로 보내면 연결 해제 (삭제 여부 별도 확인)

### 5-3. 공지 삭제 시 일정 처리

```
DELETE /api/posts/:id?delete_linked_schedule=true
```

- `delete_linked_schedule=true`: 연결 일정도 삭제
- `delete_linked_schedule=false` (기본): 일정은 연결만 해제하고 독립 유지

### 5-4. 연결 공지 조회 (일정 상세에서)

기존 `GET /api/schedules/:id` 응답에 `linked_post` 정보 추가:

```json
{
  "success": true,
  "data": {
    "id": 10,
    "title": "3월 정기 이사회",
    "linked_post": {
      "id": 15,
      "title": "3월 정기 이사회 안내",
      "author": "김회장",
      "created_at": "2026-03-10T..."
    }
  }
}
```

## 6. DB 변경사항

### 6-1. posts 테이블 컬럼 추가

```sql
-- 공지에 연결된 일정 ID
ALTER TABLE posts ADD COLUMN linked_schedule_id INTEGER REFERENCES schedules(id) ON DELETE SET NULL;
```

> 참고: `posts.schedule_id` 예비 컬럼이 이미 존재할 수 있음. 존재하면 `linked_schedule_id`로 rename 또는 기존 `schedule_id`를 활용.

### 6-2. schedules 테이블 컬럼 추가

```sql
-- 일정에 연결된 공지 ID
ALTER TABLE schedules ADD COLUMN linked_post_id INTEGER REFERENCES posts(id) ON DELETE SET NULL;
```

### 6-3. 인덱스

```sql
CREATE INDEX idx_posts_linked_schedule ON posts(linked_schedule_id) WHERE linked_schedule_id IS NOT NULL;
CREATE INDEX idx_schedules_linked_post ON schedules(linked_post_id) WHERE linked_post_id IS NOT NULL;
```

## 7. 비즈니스 규칙

1. **1:1 관계**: 하나의 공지에 하나의 일정만 연결 가능 (복수 일정 불가)
2. **단방향 생성**: 공지에서만 일정 생성 가능 (일정에서 공지 역방향 생성 불가)
3. **양방향 참조**: 공지→일정, 일정→공지 모두 참조 가능
4. **트랜잭션 보장**: 공지 + 일정 생성/수정은 하나의 DB 트랜잭션으로 처리
5. **독립 일정과의 구분**: 연결 일정은 캘린더에서 "🔗" 아이콘으로 표시 (공지 연결됨을 시각적 표시)
6. **권한**: 일정 첨부는 공지 작성 권한과 동일 (`can_post_notice = true`)
7. **투표 연결**: 일정에 투표가 첨부된 경우, 공지 상세에서도 투표 카드 표시 (일정 상세로 이동하여 투표)

## 8. 유저 플로우

### 8-1. 공지 + 일정 동시 생성 플로우

```
공지 작성 화면
    ↓
제목/내용 입력
    ↓
[일정 첨부] 토글 ON
    ↓
일정 정보 입력 (제목, 날짜, 시간, 장소, 카테고리)
    ↓
(선택) [참석 투표] 토글 ON → 투표 설정 입력
    ↓
[공지 등록] 클릭
    ↓
API: POST /api/posts (schedule + vote_config 포함)
    ↓
백엔드: 트랜잭션으로 공지 + 일정 + 투표설정 생성
    ↓
성공 → 공지 목록으로 복귀
    ↓
캘린더에 해당 일정 자동 표시
```

### 8-2. 캘린더에서 공지로 이동 플로우

```
캘린더 → 날짜 선택 → 일정 카드 클릭 (🔗 아이콘 있는 일정)
    ↓
일정 상세 화면
    ↓
[🔗 연결된 공지: "3월 정기 이사회 안내"] 클릭
    ↓
공지 상세 화면으로 이동
```

## 10. 향후 확장 가능성

- **복수 일정 연결**: 하나의 공지에 여러 일정 (예: 연속 교육 프로그램)
- **자동 리마인더**: 연결 일정의 D-1, D-day 자동 알림
- **일정에서 공지 역방향 생성**: 일정 등록 시 공지 자동 생성 옵션
