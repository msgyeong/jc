## 1. 기능 개요

일정 캘린더에서 특정 일정을 선택하면 진입하는 상세 화면. 일정 정보 전체를 표시하고, 출석투표·댓글·참석자 명단 등 부가 기능을 제공한다.

- **진입 경로**: 캘린더 날짜 선택 → 일정 카드 클릭 → 상세 화면
- **대상 사용자**: 모든 승인된 회원
- **현재 상태**: 기본 상세 보기 + 출석투표 카드가 구현되어 있음. 댓글·참석자 목록·지도 링크 등 미완성.

## 3. 섹션별 상세 스펙

### 3-1. 헤더 영역

| 요소 | 설명 |
|------|------|
| **뒤로 버튼** | 캘린더 화면으로 복귀 |
| **더보기(⋮)** | 수정/삭제 메뉴 (작성자 또는 admin만 표시) |
| **카테고리 배지** | `event`/`meeting`/`training`/`holiday`/`other` — 색상 구분 |

### 3-2. 일정 기본 정보

| 필드 | 소스 | 표시 형식 |
|------|------|-----------|
| **제목** | `title` | h2, 굵게 |
| **날짜** | `start_date`, `end_date` | `YYYY.MM.DD (요일) HH:mm` |
| **시간** | `start_date`, `end_date` | 시작~종료 (종일 이벤트는 "종일" 표시) |
| **장소** | `location` | 텍스트 + 지도 링크 아이콘 |
| **작성자** | `created_by` → users.name | 이름 표시 |
| **등록일** | `created_at` | 상대 시간 ("3일 전") |

### 3-3. 장소 — 지도 링크

```
📍 영등포구청 3층 대회의실  🗺️
```

- 장소 텍스트 옆 지도 아이콘(🗺️) 클릭 시 외부 지도 앱으로 이동
- 링크: `https://map.naver.com/v5/search/{encodeURIComponent(location)}`
- 모바일에서 네이버 지도 앱이 설치되어 있으면 앱으로 열림
- 장소가 NULL이면 장소 행 자체 미표시

### 3-4. 출석투표 카드

- **기구현 상태**: `attendance-vote.md` 기획서 기반으로 이미 구현됨
- 투표 카드, 상태별 카운트, 참석자 목록 펼침 기능 포함
- 이 기획서에서는 추가 변경 없음

### 3-5. 참석자 명단 섹션

투표 카드 아래에 "참석" 상태인 회원 명단을 별도 표시.

```
👥 참석자 명단 (15명)
김회장 · 이부회장 · 박이사 · 정이사 · 한회원 ...
[전체 보기 >]
```

- **기본 표시**: 최대 5명까지 이름 나열 + "외 N명"
- **전체 보기**: 클릭 시 전체 참석자 목록 모달 또는 펼침
  - 이름, 직책 표시
  - 투표 시점 표시 (2시간 전, 어제 등)
- **투표 미활성 일정**: 이 섹션 미표시
- **API**: 기존 `GET /api/schedules/:id/vote-details` 활용

### 3-6. 연결된 공지 (M-12 연동)

- 공지에서 생성된 일정인 경우, 원본 공지 링크 표시
- `schedules.linked_post_id`가 존재하면 표시
- 클릭 시 해당 공지 상세 화면으로 이동
- 연결 공지가 없으면 이 섹션 미표시
- 상세: `notice-schedule-link.md` 참조

### 3-7. 댓글 섹션

일정에 대한 댓글·대댓글 기능. 게시판 댓글과 동일한 구조.

| 항목 | 스펙 |
|------|------|
| **댓글 작성** | 하단 고정 입력창, 전송 버튼 |
| **댓글 목록** | 최신순 정렬, 작성자·시간·내용 표시 |
| **대댓글** | 1단계까지만 (대대댓글 불가) |
| **댓글 삭제** | 본인 댓글만 삭제 가능 (soft delete) |
| **공감** | 댓글별 👍 토글 |
| **댓글 수** | 섹션 헤더에 총 댓글 수 표시 |

## 4. 수정/삭제 기능

### 4-1. 더보기 메뉴 (⋮)

```
┌──────────────┐
│  ✏️ 수정     │
│  🗑️ 삭제     │
└──────────────┘
```

- **표시 조건**: `created_by === currentUser.id` 또는 `role === 'admin'/'super_admin'`
- 일반 회원에게는 더보기 버튼 자체 미표시

### 4-2. 수정

- 더보기 → "수정" 클릭 시 일정 등록 폼(schedule-form)으로 이동
- 기존 데이터가 폼에 프리필됨
- 수정 완료 후 상세 화면으로 복귀

### 4-3. 삭제

```
┌─────────────────────────────────┐
│  일정을 삭제하시겠습니까?        │
│  삭제 시 투표 기록도 함께        │
│  삭제됩니다.                    │
│                                 │
│  [취소]          [삭제]         │
└─────────────────────────────────┘
```

- 확인 다이얼로그 표시 후 삭제
- 삭제 시 투표 설정 + 투표 기록 CASCADE 삭제
- 삭제 후 캘린더 화면으로 복귀

## 5. 종일 이벤트 처리

| 조건 | 표시 |
|------|------|
| `start_date` 시간이 00:00이고 `end_date` 없음 | "종일" |
| `start_date` 시간이 00:00이고 `end_date` 시간이 23:59 | "종일" |
| `start_date`와 `end_date`가 다른 날짜 | "3/15(토) ~ 3/16(일)" |
| 시간이 있는 일정 | "3/15(토) 14:00 ~ 16:00" |

## 7. DB 변경사항

### 기존 테이블 활용

댓글은 기존 `comments` 테이블의 `schedule_id` 컬럼을 사용 (이미 존재).

```sql
-- comments 테이블에 이미 schedule_id 컬럼이 있음
-- 추가 변경 불필요
```

### 신규 컬럼 (공지-일정 연동용)

```sql
-- schedules 테이블에 연결 공지 ID 추가
ALTER TABLE schedules ADD COLUMN linked_post_id INTEGER REFERENCES posts(id) ON DELETE SET NULL;
```

이 컬럼은 `notice-schedule-link.md` 기획의 일부이며, 상세 화면에서 "연결된 공지" 섹션 표시에 사용.

## 9. 유저 플로우

```
캘린더 → 날짜 선택 → 일정 카드 클릭
    ↓
일정 상세 화면 진입
    ├─ 일정 정보 확인 (제목, 날짜, 장소, 설명)
    ├─ 장소 클릭 → 네이버 지도 열림
    ├─ 출석투표 참여 (참석/불참/미정)
    ├─ 참석자 명단 확인
    ├─ 연결된 공지 이동
    ├─ 댓글 읽기/작성/대댓글
    └─ (작성자/관리자) 수정 → 폼 이동 / 삭제 → 확인 후 삭제
```
