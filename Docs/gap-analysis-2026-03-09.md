## 2. 기능별 상세 갭 분석

### F-AUTH: 회원가입/로그인

| 항목 | PRD 요구사항 | 구현 상태 | 비고 |
|------|-------------|----------|------|
| 6단계 회원가입 | 필수 | ✅ 구현됨 | |
| 이메일+비밀번호 로그인 | 필수 | ✅ 구현됨 | |
| 자동 로그인 (토큰) | 필수 | ✅ 구현됨 | localStorage JWT |
| 가입 승인 대기 화면 | 필수 | ✅ 구현됨 | |
| 비밀번호 재설정 | 필수 | ⚠️ 부분 구현 | 백엔드만 존재, 프론트 미완성 가능성 |
| 카카오 소셜 로그인 | PRD 명시 | ❌ 미구현 | PRD 03-features에 명시됨 |
| 회원 상태별 접근 제어 | 필수 | ✅ 구현됨 | pending/active/suspended/rejected/withdrawn |

**프론트엔드 요청사항**: 비밀번호 재설정 화면 완성도 확인 필요
**백엔드 요청사항**: 카카오 소셜 로그인 API (MVP 후순위 가능)

## 3. 구조적 문제

### 3-1. 공지 라우트 중복
- `api/routes/notices.js` — 독립적인 공지 CRUD
- `api/routes/posts.js` (category='notice') — 게시판 내 공지 탭
- **문제**: 두 경로가 서로 다른 테이블/로직을 사용할 가능성. 데이터 불일치 위험.
- **권장**: `posts.js`의 category='notice'로 통합하고, `notices.js`는 폐기

### 3-2. PRD 기술 스택 불일치
- `Docs/prd/06-tech-stack.md`에 Flutter/Dart, Next.js 등이 명시됨
- 실제 구현은 Vanilla JS 웹앱 + Express API
- **필요 조치**: PRD 06-tech-stack.md를 현재 스택에 맞게 업데이트 필요

## 4. 미구현 기능 요약 목록

### MVP 출시에 필요한 미구현 항목
1. 배너 콘텐츠 관리 및 표시 로직
2. 공지 라우트 통합 (notices.js 정리)
3. 회원 상세 프로필 보기 (타인)
4. 일정 상세/등록/수정 화면 완성
5. N배지 하단 탭 적용
6. 빈 상태/로딩/에러 UI 전체 적용
7. 관리자 웹 기본 기능 (회원 승인)

### MVP 이후 구현 가능 항목
1. 카카오 소셜 로그인
2. 고급 회원 검색 (직업/업종/직책/기수 필터)
3. 출석 조사 전체 기능
4. Cloudinary 이미지 스토리지
5. 푸시 알림
6. 알림 센터
7. 회원 정지/해제 관리
8. 일정-게시글 연동
