## 1. 현재 홈 화면 구현 상태

```
┌─────────────────────────────────┐
│  영등포JC          (알림 벨 없음) │  ← AppBar
├─────────────────────────────────┤
│  [배너 슬라이더]                  │  ← 구조만 존재, 콘텐츠 없음
├─────────────────────────────────┤
│  📋 공지사항           전체 보기 → │
│  ┌─────────────────────────────┐│
│  │ 공지 제목 1        2026-03-09││
│  │ 공지 제목 2        2026-03-08││
│  │ ... (최대 5개, 7일 필터)     ││
│  └─────────────────────────────┘│
├─────────────────────────────────┤
│  📅 다가오는 일정      전체 보기 → │
│  ┌─────────────────────────────┐│
│  │ 일정 제목 1   03/15 14:00   ││
│  │ 일정 제목 2   03/20 10:00   ││
│  │ ... (최대 5개)              ││
│  └─────────────────────────────┘│
├─────────────────────────────────┤
│  홈 │ 게시판 │ 일정 │ 회원 │ 프로필 │
└─────────────────────────────────┘
```

**문제점**:
1. 배너 영역에 표시할 콘텐츠가 정의되지 않음
2. 배너 관리(등록/수정/삭제) 방법이 없음
3. 알림 벨 아이콘 미구현
4. 공지/일정 카드에 N배지 미적용

## 2. 배너 콘텐츠 정의

### 2-1. 배너에 보여줄 콘텐츠 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **회원 업체 홍보** | 회원이 운영하는 사업체 광고 | "OO법률사무소 — 회원 특별 상담" |
| **JC 행사 안내** | 중요 행사/교육 배너 형태 안내 | "제42기 입회식 — 3월 25일" |
| **협력 업체 광고** | 영등포JC 후원/협력사 홍보 | "XX은행 JC회원 전용 금리" |
| **공지 하이라이트** | 중요 공지를 배너로 강조 | "2026년 연회비 납부 안내" |

### 2-2. MVP 단계 배너 운영 방안

MVP에서는 **관리자가 직접 배너를 등록**하는 가장 단순한 구조를 사용한다.

```
배너 데이터 구조:
{
  id: number,
  title: string,           // 배너 제목 (업체명 또는 행사명)
  subtitle: string,        // 짧은 홍보 문구
  image_url: string,       // 배너 이미지 URL (선택)
  link_url: string,        // 클릭 시 이동할 URL (선택, 외부 링크 가능)
  link_type: 'external' | 'notice' | 'schedule',  // 링크 유형
  link_target_id: number,  // 내부 링크일 경우 대상 ID
  bg_color: string,        // 배경색 (이미지 없을 때)
  is_active: boolean,      // 노출 여부
  display_order: number,   // 표시 순서
  start_date: date,        // 노출 시작일
  end_date: date,          // 노출 종료일
  created_by: number,      // 등록 관리자
  created_at: timestamp
}
```

### 2-3. 배너 운영 규칙

- 동시 노출 배너: 최소 1개, 최대 5개
- 자동 슬라이드: 4초 간격
- 수동 스와이프: 좌우 가능
- 배너 없을 때: 영역 자체를 숨김 (빈 공간 방지)
- 노출 기간: start_date ~ end_date 범위 내 자동 노출/숨김
- 등록 권한: 관리자(super_admin, admin)만 가능

### 2-4. 배너 클릭 동작

| link_type | 동작 |
|-----------|------|
| `external` | 새 탭/브라우저에서 link_url 열기 |
| `notice` | 앱 내 공지 상세 화면으로 이동 (link_target_id) |
| `schedule` | 앱 내 일정 상세 화면으로 이동 (link_target_id) |

## 3. 홈 화면 섹션 구성 재정의

### 3-1. 최종 홈 화면 레이아웃

```
┌─────────────────────────────────┐
│  영등포JC                    🔔 │  ← AppBar (알림 벨, MVP 후순위)
├─────────────────────────────────┤
│                                 │
│  ┌─ 배너 섹션 ────────────────┐ │
│  │  [이미지 또는 컬러 배경]     │ │
│  │  업체명/행사명              │ │
│  │  짧은 홍보 문구             │ │
│  │         ● ○ ○ (인디케이터)  │ │
│  └────────────────────────────┘ │
│                                 │
│  ┌─ 공지사항 ─────────────────┐ │
│  │  📋 공지사항      전체 보기 →│ │
│  │  ─────────────────────────  │ │
│  │  📌 [고정] 제42기 입회식...  │ │  ← 고정 공지 우선
│  │  🔵 연회비 납부 안내    N   │ │  ← N배지 (미읽음)
│  │  회장단 회의 결과 공유       │ │
│  │  3월 교육 일정 안내         │ │
│  │  신규 회원 환영 인사         │ │
│  └────────────────────────────┘ │
│                                 │
│  ┌─ 다가오는 일정 ────────────┐ │
│  │  📅 다가오는 일정  전체 보기 →│ │
│  │  ─────────────────────────  │ │
│  │  🔴 3/15(토) 정기 총회      │ │  ← 카테고리 컬러 도트
│  │  🟡 3/20(목) 신입회원 교육  │ │
│  │  🔵 3/25(화) 이사회         │ │
│  │  🟢 4/01(화) 봉사활동       │ │
│  │  🔴 4/05(토) 체육대회       │ │
│  └────────────────────────────┘ │
│                                 │
│  ┌─ 빠른 액션 ───────────────┐ │  ← 신규 섹션
│  │  [회원 검색]  [일정 등록]   │ │
│  └────────────────────────────┘ │
│                                 │
├─────────────────────────────────┤
│  홈 │ 게시판 │ 일정 │ 회원 │ 프로필 │
└─────────────────────────────────┘
```

### 3-2. 섹션별 상세 정의

#### 섹션 1: AppBar
- 좌측: 앱 로고 또는 "영등포JC" 텍스트
- 우측: 알림 벨 아이콘 (MVP에서는 비활성 또는 미표시, v1.0에서 활성화)

#### 섹션 2: 배너 슬라이더
- 위치: 스크롤 영역 최상단
- 높이: 160px (고정)
- 콘텐츠: 관리자가 등록한 활성 배너 (2-2 참조)
- 배너 0개일 때: **섹션 자체를 숨김**
- 인디케이터: 하단 중앙 도트

#### 섹션 3: 공지사항 요약
- 헤더: "공지사항" + "전체 보기" 버튼
- 카드 수: 최대 5개
- 정렬: 고정 공지(📌) 우선 → 최신순
- 필터: 최근 7일 이내 (현행 유지)
- 카드 정보: 제목, 등록일, 고정 표시, N배지(미읽음)
- 카드 클릭 → 공지 상세 화면 이동
- "전체 보기" → 게시판 화면 공지 탭으로 이동
- 공지 0개일 때: "등록된 공지가 없습니다" 메시지

#### 섹션 4: 다가오는 일정 요약
- 헤더: "다가오는 일정" + "전체 보기" 버튼
- 카드 수: 최대 5개
- 정렬: 날짜 오름차순 (가장 가까운 일정 먼저)
- 필터: 오늘 이후 일정만
- 카드 정보: 카테고리 컬러 도트, 날짜(요일), 제목, 시간, 장소
- 카드 클릭 → 일정 상세 화면 이동
- "전체 보기" → 캘린더 화면으로 이동
- 일정 0개일 때: "예정된 일정이 없습니다" 메시지

#### 섹션 5: 빠른 액션 (신규 추가)
- 목적: 자주 사용하는 기능으로의 바로가기
- 버튼 구성:
  - "회원 검색" → 회원 화면 이동
  - "일정 등록" → 일정 등록 화면 (관리자에게만 표시)
- 스타일: 아웃라인 버튼, 가로 나란히 배치
- **MVP 선택사항**: 없어도 출시 가능. 사용 패턴 보고 판단.

## 4. 백엔드/프론트엔드 요청사항

### 백엔드 에이전트에게
1. **banners 테이블 생성** (위 2-2 데이터 구조 참조)
2. **배너 API 구현**:
   - `GET /api/banners` — 활성 배너 목록 (is_active=true, 기간 내)
   - `POST /api/admin/banners` — 배너 등록 (관리자)
   - `PUT /api/admin/banners/:id` — 배너 수정
   - `DELETE /api/admin/banners/:id` — 배너 삭제
3. **홈 통합 API** (선택): `GET /api/home` — 배너 + 공지 5개 + 일정 5개 한번에 반환

### 프론트엔드 에이전트에게
1. **배너 슬라이더 구현**: 자동/수동 슬라이드, 인디케이터, 클릭 이동
2. **공지 카드 N배지 추가**: 미읽음 공지에 "N" 표시
3. **일정 카드 카테고리 컬러 도트 추가**
4. **빈 상태 메시지**: 각 섹션 데이터 없을 때 안내 문구
5. **배너 0개 시 섹션 숨김 처리**

### 디자이너 에이전트에게
1. 배너 슬라이더 상세 디자인 (높이, 텍스트 배치, 오버레이 등)
2. 빠른 액션 섹션 디자인 (필요 시)
