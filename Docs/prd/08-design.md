# PRD - 설계

영등포 JC 회원관리 앱 설계 문서

본 문서는 제품의 목표, 범위, 정책, 기능 정의를 포함하며,
**유저 플로우, 화면 구조, 기능 상세 설계**를 기반으로 개발 착수를
가능한 상태를 목표로 한다.

---

## 1. 제품 개요

### 1.1 제품 목적
- 영등포 JC 회원 간의 공지 전달, 소통, 일정 공유, 회원 정보 관리를 하나의 앱에서 제공한다.
- 오프라인 중심 커뮤니티 운영을 온라인으로 보조한다.

### 1.2 사용자 정의

- 회원
  - 모든 회원은 동일한 사용자 경험을 가진다.
  - 특우회 여부는 **프로필 정보로만 구분**된다.
  - 기능 접근 권한의 기준이 아니다.

- 최고 관리자
  - `manager_###` 계정으로 **관리자 웹**에서 접근하는 최상위 권한 주체
  - 가입 승인/회원 관리/직책 및 권한 부여·회수 수행

- 앱 내 관리자(직책 기반)
  - 관리자 웹에서 특정 직책/권한을 부여받아 **앱**에서 권한 기능을 수행하는 회원
  - 공지사항 작성, 일정 등록 등

## 4. 화면 구조 정의 (S)

### 4.1 전체 화면 요약

- 코어 화면: 11개 (S-001 ~ S-011)
- 관리자 전용 화면: 3개 (S-012 ~ S-014)
- 확장 화면: 0~2개 (S-015 ~ S-016)

---

### 4.2 공통 UI 규칙

#### 4.2.1 새 콘텐츠 표시 ('N' 배지)

- **적용 대상**
  - 게시글 (목록/홈 요약)
  - 공지사항 (목록/홈 요약)
  - 일정 (목록/캘린더/홈 요약)

- **표시 조건 (AND 조건)**
  - 게시글/공지사항:
    - 생성일로부터 3일 이내
    - 사용자가 해당 콘텐츠를 읽지 않음
  - 일정:
    - 일정 시작일이 지나지 않음
    - 생성일로부터 3일 이내
    - 사용자가 해당 콘텐츠를 읽지 않음

- **UI 표현**
  - 'N' 배지 또는 'new' 라벨 표시
  - 위치: 제목 옆 또는 카드 우측 상단
  - 색상: 강조 색상(design-system/01-color-palette.md 참조)

- **동작**
  - 사용자가 상세 화면 진입 시 해당 콘텐츠의 'N' 표시 제거
  - 읽음 상태는 사용자별로 개별 관리
  - 읽음 처리는 서버에 기록하여 다른 기기에서도 동기화

- **기술 요구사항**
  - 사용자별 읽음 여부 추적 테이블 필요
  - 생성일 기준 3일 경과 시 자동으로 'N' 표시 제거(읽음 여부 무관)
  - 일정의 경우 일정 시작일 경과 시 자동으로 'N' 표시 제거

## 5. 주요 기능 상세 설계

> 기능 상세 설계는 본 문서 기준으로 관리하며,
> 다른 PRD 문서에는 상세를 중복 기재하지 않는다.
>
> **유저 플로우 상세**: 각 기능의 "관련 유저 플로우" 섹션에서 플로우 ID를 참조하며,
> 상세 내용은 `../user-flows/` 폴더의 문서를 참조하세요.

### 5.1 설계 순서

설계 순서는 줌인/줌아웃 원칙에 따라 다음과 같이 진행한다.
1. 인증 시스템 (로그인/회원 가입)
2. 게시글 관련 기능
3. 공지사항 관련 기능
4. 일정 관련 기능
5. 홈 화면 관련 기능 묶음
6. 회원/프로필 관련 기능
7. 관리자 전용 기능

---

### 5.2 기능 상세 설계 반영 요약

#### 5.2.1 게시글

- 목록 → 상세 → 작성 흐름 일관 유지
- 작성은 전체 회원 가능
- 수정/삭제는 작성자 또는 **앱 내 관리자(직책 기반)**만 가능

#### 5.2.2 공지사항

- 게시글과 구조적으로 분리
- 고정 공지 우선 노출
- 작성은 **앱 내 관리자(직책 기반)**만 가능

#### 5.2.3 일정

- 날짜 중심 탐색
- 홈 요약 → 일정 화면 → 상세 흐름
- 등록은 **앱 내 관리자(직책 기반)**만 가능

#### 5.2.4 회원 / 프로필

- 회원 간 기능 차별 없음
- 특우회 여부는 정보 표시용
- 회원 관리 액션(승인/직책/권한 등)은 **관리자 웹**에서만 가능(최고 관리자 수행)

## F-AUTH. 인증 시스템 (로그인/회원 가입)

- 기능 개요  
  사용자가 앱에 로그인하고 회원 가입을 할 수 있도록 한다.

- 포함 UI 요소  
  - 로그인 화면
  - 회원 가입 화면 (회원 기록 카드 기준)
    - 첫 사용(온보딩) 관점의 UX 요소
      - 상단 진행 표시(예: 1/3) 및 현재 단계 안내
      - 필수/선택 항목 라벨, 입력 예시(placeholder) 및 도움말 텍스트
      - 각 입력 항목 하단의 오류 메시지 영역(스낵바 대신 인라인)
      - 단계형 구성 시: 이전/다음/임시 저장(자동) 버튼
      - 제출(가입 승인) 시: 로딩/비활성 상태, 중복 제출 방지
    - 로그인 정보 입력 필드
      - E-mail (필수)
      - 비밀번호 8자리 (필수)
      - 비밀번호 확인 8자리 (필수)
    - 기본 정보 입력 필드
      - 프로필 사진 업로드 (1매 필수, 500mb 이하)
      - 성명 (한글 필수, 한자/영문 선택)
      - 주민등록번호 (하이픈 포함)
      - 자택 주소 (상세 주소 포함)
      - 휴대폰 (하이픈 포함)
    - 입력 편의/표준 패턴(일반 앱 수준)
      - 비밀번호: 표시/숨김 토글, 규칙 안내, 불일치 즉시 안내
      - 주민등록번호: 숫자 키패드, 자동 하이픈, 입력 마스킹 표시
      - 휴대폰: 숫자 키패드, 자동 하이픈
      - 주소: 검색/자동완성(가능 시) 또는 입력 가이드 제공
      - 프로필 사진: 권한 요청 → 선택/촬영 → 크롭/미리보기 → 업로드 진행 표시
      - 네트워크 실패: 재시도 버튼 및 실패 사유 인라인 표시
    - 직장 정보 입력 필드 (선택)
      - 직장명
      - 직위
      - 직장 주소
      - 직종
    - 학력 정보 입력 필드
      - 학력 (최소 1개 이상)
      - 졸업 년월 (6자리)
      - 학교명 (필수)
    - 경력 정보 입력 필드
      - 경력 (최소 1개 이상)
      - 경력 년월 (6자리)
      - 경력사항 (필수)
    - 가족 정보 입력 필드
      - 혼인 유무 (필수)
      - 혼인인 경우 (필수)
        - 배우자명
        - 배우자 연락처 (하이픈 포함)
        - 배우자 생년월일 (8자리)
      - 자녀 유무 (필수)
      - 자녀 있는 경우 (필수)
        - 자녀 이름 (자녀별 추가 입력)
    - 기타 정보 입력 필드 (선택)
      - 취미
      - 특기
      - 추천인
    - 프로필 사진 업로드 UI (이미지 선택, 미리보기, 1매 필수)
    - 개인정보 보호 동의 체크박스
    - 가입 승인 버튼
  - e-mail/비밀번호 입력 필드

- 기능 요구 사항  
  - **회원 가입** (회원 기록 카드 기준)
    - 필수 입력 항목:
      - E-mail
      - 비밀번호 8자리
      - 비밀번호 확인 8자리
      - 프로필 사진 (1매 필수, 500mb 이하)
      - 성명 (한글 필수, 한자/영문 선택)
      - 주민등록번호 (하이픈 포함)
      - 자택 주소 (상세 주소 포함)
      - 휴대폰 (하이픈 포함)
      - 학력 (최소 1개 이상: 졸업 년월 6자리, 학교명)
      - 경력 (최소 1개 이상: 경력 년월 6자리, 경력사항)
      - 혼인 유무 (혼인인 경우: 배우자명, 배우자 연락처, 배우자 생년월일)
      - 자녀 유무 (자녀 있는 경우: 자녀 이름)
    - 선택 입력 항목:
      - 직장명, 직위, 직장 주소, 직종
      - 취미, 특기, 추천인
    - 프로필 사진 업로드 (1매 필수)
      - 이미지 선택 및 미리보기
      - Supabase Storage에 업로드
      - 프로필 사진 URL 저장
    - E-mail에 key 값 부여
    - 이메일 입력 UX(일반 앱 수준)
      - 이메일 형식 즉시 검증(입력 종료 시)
      - 중복 이메일(이미 가입된 이메일) 안내를 인라인으로 제공
    - 필수 입력 완료 후에만 "가입 승인" 버튼 활성화
    - 가입 폼 UX(첫 사용 경험 최적화)
      - 오류는 해당 입력 항목 하단에 표시(스낵바 대신 인라인)
      - 오류가 있어도 맞게 입력된 정보는 유지(재입력 최소화)
      - 단계형 UI인 경우, 다음 단계 이동 시 미완료 필수 항목으로 포커스 이동
      - 제출 중에는 버튼 비활성 + 진행 표시로 중복 제출 방지
      - 네트워크/업로드 실패 시 재시도 제공(입력값 유지)
    - 입력 형식 오류 시 해당 항목 아래에 오류 메시지 표시
    - 오류가 있어도 맞게 입력된 정보는 유지
    - 개인정보 보호 동의 필수
    - 회원 가입 시 `is_approved = false` (가입대기 상태)로 생성
    - 회원 가입 완료 후 플로우
      - "승인 요청" 버튼 클릭 → 회원가입 정보 저장
      - 다음 화면: "회원 가입이 승인 진행중입니다" + '확인' 버튼
      - 확인 버튼 클릭 → 로그인 화면으로 복귀
  - **스플래시 화면**
    - 앱 실행 시 `assets/images/jc_logo_slash.png` 이미지 표시
    - 서버에서 데이터를 가져오는 것을 완료했을 때까지 표시 (최대 3초)
    - 인증 상태 확인 후 로그인 화면 또는 홈 화면으로 전환
  - **로그인**
    - 로그인 화면 구성
      - (원칙) 로그인 화면에서 바로 이메일/비밀번호 입력 가능해야 한다.
      - 이메일 입력 필드
      - 비밀번호 입력 필드
      - 로그인 버튼
      - 로그인 유지(이메일/비밀번호 기억하기 또는 자동 로그인) 옵션
      - 하단 버튼: "아이디/비밀번호 찾기"(이메일 기반 비밀번호 재설정), "회원가입"
    - 아이디 및 비밀번호 로그인
    - 소셜 로그인 지원
      - 카카오 로그인
    - 소셜 로그인 사용 시 Supabase Authentication과 연동
    - 소셜 로그인 후 회원 기록 카드 필수 정보 입력 필요
    - **주의**: 소속 JC, JC 직책은 관리자 웹에서만 입력/수정 가능
    - **가입 승인된 회원만 로그인 가능** (`is_approved = true`, `is_suspended = false`, `withdrawn_at IS NULL`)
    - **로그인 에러 처리**
      - 미등록 이메일: "등록되지 않은 회원입니다" 메시지 표시
      - 비밀번호 불일치: "비밀번호가 일치하지 않습니다" 메시지 표시 (정중한 톤)
    - **회원 상태별 로그인 처리**
      - 가입대기 상태 (`is_approved = false`, `rejection_reason IS NULL`): 승인 진행중 화면으로 리다이렉트
      - 거절 상태 (`is_approved = false`, `rejection_reason IS NOT NULL`): 로그인 차단, 거절 사유 안내
      - 정지 상태 (`is_suspended = true`): 로그인 차단, 정지 안내
      - 탈퇴 상태 (`withdrawn_at IS NOT NULL` 또는 `is_deleted = true`): 로그인 차단
    - **승인 진행중 화면**
      - 로그인 시도 시에만 접근 가능 (일반 네비게이션 불가)
      - "회원 가입이 승인 진행중입니다" 메시지 표시
      - "확인" 버튼만 제공 (로그인 화면으로 복귀)
    - **자동 로그인 유지 기능**
      - 한번 로그인하면 해당 기기에서 계속 로그인 상태 유지
      - 앱 삭제 또는 명시적 로그아웃 시에만 로그인 상태 해제
      - 핸드폰 변경 시에만 다시 로그인 필요
    - 로그인 상태 관리 (Riverpod)
  - **로그아웃**
    - 로그아웃 버튼 제공
    - Supabase 세션 해제
    - 로컬 저장된 로그인 상태 삭제
    - 로그인 화면으로 이동
  - **로그인하지 않은 사용자 접근 제어** ⚠️ **필수**
    - 로그인하지 않은 상태에서는 앱의 모든 기능에 접근 불가
    - 하단 탭 네비게이션 숨김
    - 모든 보호된 화면 접근 차단 (홈, 게시판, 일정, 회원, 내 프로필)
    - 로그인 화면으로 자동 리다이렉트
    - 인증 가드를 통한 접근 제어 필수

- 관련 유저 플로우  
  - UF-001 앱 진입 및 홈 화면 진입 (`../user-flows/01-app-flows.md#uf-001-앱-진입-및-홈-화면-진입` 참조)

## F-011. 프로필 상세 조회 및 내 프로필 관리

- 기능 개요  
  회원의 기본 정보를 확인하고, 자신의 프로필 정보를 수정한다. 단, 관리자 웹에서 지정한 특정 항목은 수정할 수 없으며, 관리자가 입력한 사항은 자동으로 공개된다.

- 포함 UI 요소  
  - 이름
  - 소속/기수
  - 특우회 여부 표시
  - 기타 PRD 정의 정보
  - 프로필 수정 폼 (내 프로필인 경우)
  - 관리자 전용 필드 표시 (읽기 전용)

- 기능 요구 사항  
  - **프로필 상세 조회**
    - 읽기 전용 제공
    - 관리자는 관리 기능 접근 가능
  - **내 프로필 수정**
    - 프로필 정보 수정 가능
    - **관리자 전용 필드는 수정 불가** (읽기 전용)
      - 소속 JC
      - 회원번호
      - JC 직책
      - 가입일
      - 소속분과
      - 전입 (년, 월, 일)
      - 연수 (신입/기존/임원/기타 - 각각 년/월/일, 기수)
      - 표창 (최대 5개)
      - 국제대회 참가 (최대 2개)
      - JC경력 (최대 5개)
      - 징계
    - 관리자 전용 필드 수정 시도 시 에러 메시지 표시
    - 프로필 사진 업로드 가능
  - **자동 공개 정책**
    - 관리자가 입력한 다음 항목은 프로필에서 자동 공개:
      - 소속 JC
      - JC 직책
      - 가입일
      - 연수
      - 표창
      - 국제대회 참가
      - JC경력
    - 관리자가 입력하지 않은 항목은 비공개

- 관련 유저 플로우  
  - UF-007 회원 검색 및 프로필 조회 (`../user-flows/01-app-flows.md#uf-007-회원-검색-및-프로필-조회` 참조)

  - 참석자 목록 표시
  - 참석자 통계 표시 (참석/불참/미정 수)

- 기능 요구 사항  
  - 공지사항 작성 시 참석자 조사 활성화 옵션 제공
  - 참석자 조사 활성화된 공지사항에만 참석자 조사 UI 표시
  - 회원은 자신의 참석 여부를 선택/변경 가능
  - 참석자 목록은 공개적으로 표시
  - 참석자 통계는 실시간으로 업데이트
  - 참석 여부 변경 시 즉시 반영

- 관련 유저 플로우  
  - UF-002 홈 화면에서 주요 콘텐츠 탐색 (`../user-flows/01-app-flows.md#uf-002-홈-화면에서-주요-콘텐츠-탐색` 참조)
  - UF-004 공지 게시글 작성 (참석자 조사 활성화 옵션 포함) (`../user-flows/01-app-flows.md#uf-004-공지-게시글-작성` 참조)
