```
[1주차] M-02(공지 통합) → M-12(공지-일정 연동) → M-04, M-05(일정 완성)
[2주차] M-03(회원 상세+바로통화) → M-13(업종 검색) → M-14(직함 관리)
        M-17(즐겨찾기) → M-16(바로 통화)
[3주차] M-11(참석 투표) → M-06(N배지) → M-07(상태 UI) → M-01(배너)
[4주차] M-08a(기존 삭제) → M-08b~c(로그인+승인) → M-08d~e(회원목록+카드)
[5주차] M-08f(직책/권한) → M-08g~i(게시판/공지/일정) → M-08j(보안) → M-10
[6주차] 통합 테스트 → 배포 검증 → 🚀 MVP 출시
[7주차~] Should-have 순차 진행 (S-02, S-07, S-08, S-09, S-11 우선)
```

### 병렬 진행 가능 트랙

```
트랙 A (앱 모바일 웹):    M-02 → M-12 → M-03/M-16/M-17 → M-11 → M-06/M-07
트랙 B (관리자 웹):       M-08a → M-08b~c → M-08d~f → M-08g~j
트랙 C (백엔드 공통):     M-14 → M-10 → M-13 → Audit Log 테이블
```

프론트엔드 에이전트가 트랙 A와 B를 순차 진행하거나, 두 에이전트로 분담하면 4~5주에 완료 가능.
