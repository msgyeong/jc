## 핵심 원칙

### 1. 사용자 평등
- 모든 회원은 동일한 사용자 경험을 가진다
- 특우회 여부는 프로필 정보로만 구분되며, 기능 접근 권한의 기준이 아니다

### 2. 권한 관리
- 최고 관리자: `manager_###` 계정으로 관리자 웹에서 접근
- 앱 내 관리자(직책 기반): 관리자 웹에서 특정 직책/권한을 부여받아 앱에서 권한 기능 수행

### 3. 디자인 시스템
- **라이트 테마만 사용** (다크 테마 미사용)
- 신뢰감, 정돈됨, 공식성을 우선
- 정보 위계 명확히 드러나야 함
- 중립 색상 + 절제된 포인트

## 주요 기능 요약

- **인증 시스템**: 로그인/회원가입, 가입 승인 프로세스
- **게시판 구분**
  - **공지사항 게시판**: 관리자 및 관리자가 지정한 사람만 작성. 일정 첨부 가능.
  - **일반 게시판**: 모든 승인된 회원 작성. 일정 첨부 가능.
- **게시글**: 목록/상세/작성, 댓글/대댓글, 공감, **일정 첨부**(추후 일정과 연동)
- **일정 연동(추후)**: 푸시 알람(예: 1주일 전·1일 전), 참석 인원 옵션(참석자 확인)
- **공지사항**: 목록/상세/작성(권한 필요), 참석자 조사
- **일정**: 목록/상세/등록(권한 필요)
- **회원/프로필**: 목록/검색, 상세 조회, 내 프로필 수정
- **홈 화면**: 배너, **일정 첨부 게시글 최대 3개**(기간 지난 것 미노출), 공지·일정 요약. **N배지**: 새 글 표시, 읽은 사용자에게는 N 미표시(개인별 읽음)
- **관리자 웹**: 회원 관리, 권한 관리, 콘텐츠 관리
