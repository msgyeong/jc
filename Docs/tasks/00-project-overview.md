# 프로젝트 개요 및 개발 원칙

**프로젝트명**: 영등포 JC 회원관리 앱  
**목적**: 영등포 JC 회원 간의 공지 전달, 소통, 일정 공유, 회원 정보 관리를 하나의 앱에서 제공하며, 오프라인 중심 커뮤니티 운영을 온라인으로 보조

---

## 핵심 원칙

### 1. 사용자 평등
- 모든 회원은 동일한 사용자 경험을 가진다
- 특우회 여부는 프로필 정보로만 구분되며, 기능 접근 권한의 기준이 아니다

### 2. 권한 관리
- 최고 관리자: `manager_###` 계정으로 관리자 웹에서 접근
- 앱 내 관리자(직책 기반): 관리자 웹에서 특정 직책/권한을 부여받아 앱에서 권한 기능 수행

### 3. 디자인 시스템
- **라이트 테마만 사용** (다크 테마 미사용)
- 신뢰감, 정돈됨, 공식성을 우선
- 정보 위계 명확히 드러나야 함
- 중립 색상 + 절제된 포인트

---

## 기술 스택

### 앱 (Flutter)
- Flutter, Dart
- Riverpod (상태 관리)
- Freezed (불변 모델)
- Supabase (백엔드)
- GoRouter (라우팅)

### 관리자 웹 (Next.js)
- Next.js 14+ (App Router)
- TypeScript
- Zustand (상태 관리)
- Tailwind CSS, shadcn/ui

### 백엔드
- Supabase (PostgreSQL)
- Supabase Authentication
- Supabase Storage
- Supabase Realtime

---

## 주요 기능 요약

- **인증 시스템**: 로그인/회원가입, 가입 승인 프로세스
- **게시글**: 목록/상세/작성, 댓글/대댓글, 공감
- **공지사항**: 목록/상세/작성(권한 필요), 참석자 조사
- **일정**: 목록/상세/등록(권한 필요)
- **회원/프로필**: 목록/검색, 상세 조회, 내 프로필 수정
- **홈 화면**: 배너, 공지사항 요약, 일정 요약
- **관리자 웹**: 회원 관리, 권한 관리, 콘텐츠 관리
