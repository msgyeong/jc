## 유저 플로우 분류

### 플랫폼 기준 분리

본 프로젝트는 **플랫폼 기준으로 유저 플로우를 분리**하여 관리합니다.

**이유:**
- 개발 팀 구조와 일치 (앱 개발팀 vs 웹 개발팀)
- 문서 유지보수 효율성 향상
- 각 플랫폼의 특성에 맞는 상세도로 작성 가능
- 플랫폼 간 연동 지점을 명시적으로 관리

### 플로우 ID 규칙

- **앱 플로우**: `UF-###` (예: UF-001, UF-002)
- **관리자 웹 플로우**: `UF-AW-###` (예: UF-AW-001, UF-AW-002)
- **시스템/공통 플로우**: `UF-SYS-###` (예: UF-SYS-001, UF-SYS-002)

## 문서 구조화 배경

### 이전 구조

- 유저 플로우가 여러 문서에 분산 (PRD, wireframes, admin-web)
- 플랫폼 간 연동 지점이 명확하지 않음
- 개발자가 필요한 플로우를 찾기 어려움

### 개선된 구조

- **플랫폼별로 명확히 분리**하여 개발 팀이 필요한 문서만 참조
- **플랫폼 간 연동 지점**을 명시적으로 문서화 (`flows-mapping.md`)
- **공통 로직**을 별도 관리하여 중복 제거 (`03-system-flows.md`)

자세한 분석은 `../user-flows-analysis.md` 참조
