# 홈 화면 텍스트 와이어프레임

---

## [홈 화면]

### 화면 목적
- 앱 진입 시 사용자가 가장 중요한 정보를 한눈에 파악한다.
- 공지사항과 다가오는 일정을 우선적으로 전달한다.
- 주요 상세 화면으로의 진입점을 제공한다.
- 추후 확장 가능한 홍보/광고 영역의 구조를 포함한다.

---

## 상단 영역

### 구성 요소
- 앱 이름 또는 로고
- 알림 진입 버튼('종'모양, 새 알람 존재 시 'N' 표시)

### 위치
- 화면 상단 고정 영역

### 역할
- 앱의 정체성 표시
- 알림으로의 빠른 진입 제공
- 모든 화면에서 공통적으로 유지되는 기준점

---

## 중앙 영역 (스크롤 영역)

### 1. 홍보/광고 배너 영역

#### 구성 요소
- 가로 슬라이드 배너 (1~3개)
  - 업체명 또는 제목
  - 대표 이미지 또는 로고
  - 짧은 홍보 문구

#### 위치
- 중앙 영역 최상단

#### 역할
- 회원 업체 또는 협력 업체 홍보 공간
- 홈 레이아웃을 깨지 않도록 독립 섹션으로 구성

---

### 2. 공지사항 요약 영역

#### 구성 요소
- 공지사항 카드 리스트 (5개)
  - 공지 제목
  - 등록일 또는 고정 표시
- "전체 공지 보기" 버튼

#### 위치
- 홍보/광고 배너 하단

#### 역할
- 전체 회원이 반드시 확인해야 할 정보를 우선 전달
- 공지 상세 및 공지 목록 화면으로의 진입 제공

---

### 3. 다가오는 일정 요약 영역

#### 구성 요소
- 일정 카드 리스트 (5개)
  - 일정 제목
  - 날짜
  - 시간(존재 시)
  - 장소(존재 시)
- "캘린더 보기" 또는 "전체 일정 보기" 버튼

#### 위치
- 공지사항 요약 영역 하단

#### 역할
- 가까운 일정 위주로 노출하여 참여 판단을 돕는다
- 일정 상세 및 일정/캘린더 화면으로의 진입 제공

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션 (5개 탭)
  - 홈
  - 게시판
  - 일정
  - 회원
  - 내 프로필

### 위치
- 화면 하단 고정 영역

### 역할
- 앱 내 주요 화면 간 이동 제공
- 홈 화면을 기준으로 한 전체 구조 유지

---

## 홈 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 홈 | 상단/중앙/하단 | 앱 실행 또는 홈 탭 선택 | 홈 화면 구성 요소 로딩 |
| 2 | 홈 | 공지 카드 | 공지 선택 | 공지 상세 화면으로 이동 |
| 3 | 홈 | 공지 전체 보기 | 버튼 선택 | 공지 목록 화면으로 이동 |
| 4 | 홈 | 일정 카드 | 일정 선택 | 일정 상세 화면으로 이동 |
| 5 | 홈 | 일정 전체 보기 | 버튼 선택 | 일정/캘린더 화면으로 이동 |
| 6 | 홈 | 홍보 카드 | 카드 스와이프/선택 | 홍보 상세 또는 외부 링크 이동 |
| 7 | 홈 | 하단 탭 | 탭 선택 | 해당 탭 화면으로 이동 |

---

## 설계 메모

- 홈 화면에는 상세 소비용 콘텐츠를 직접 배치하지 않는다.
- 모든 카드형 콘텐츠는 "요약 + 진입" 역할만 수행한다.
- 공지사항과 일정은 홈의 핵심 정보로 항상 상단 우선 배치한다.
- 홍보/광고 영역은 기능 확장을 고려한 구조만 선반영한다.
