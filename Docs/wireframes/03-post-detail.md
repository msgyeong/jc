## [게시글 상세 화면]

### 화면 목적
- 선택한 게시글의 전체 내용을 읽는다.
- 게시글의 맥락(작성자, 작성 시점)을 명확히 인지한다.
- 작성자 또는 관리자의 경우 관리 액션으로 진입할 수 있도록 한다.
- 게시글 목록으로 자연스럽게 복귀할 수 있도록 한다.

---

## 상단 영역

### 구성 요소
- 뒤로 버튼
- 화면 제목 또는 게시글 제목 요약
- 옵션 메뉴 버튼
  - 수정(권한 보유 시)
  - 삭제(권한 보유 시)

### 위치
- 화면 상단 고정 영역

### 역할
- 이전 화면(게시글 목록)으로의 복귀 제공
- 게시글 관리 액션 진입 제공(권한 기반 노출)

---

## 중앙 영역 (스크롤 영역)

### 게시글 메타 정보 영역

#### 구성 요소
- 게시글 제목
- 작성자 이름
- 작성일

#### 위치
- 중앙 영역 상단

#### 역할
- 게시글의 기본 정보 전달
- 콘텐츠 신뢰도 및 시점 인지

---

### 게시글 본문 영역

#### 구성 요소
- 본문 텍스트
- 줄바꿈 및 문단 구조 유지

#### 위치
- 게시글 메타 정보 영역 하단

#### 역할
- 게시글의 핵심 콘텐츠 제공

---

### 첨부 이미지 영역 (존재 시)

#### 구성 요소
- 이미지 리스트 (최대 5개)
- 이미지 미리보기 (갤러리 형식)

#### 위치
- 본문 영역 하단

#### 역할
- 게시글 보조 콘텐츠 제공
- 이미지 확대 또는 전체 보기로의 진입 가능

---

### 공감 영역

#### 구성 요소
- 공감 버튼 (아이콘 + 숫자)
- 공감한 사용자 목록(선택 시 표시)

#### 위치
- 첨부 이미지 영역 하단 (또는 본문 하단)

#### 역할
- 게시글에 대한 긍정 반응 제공
- 공감 수 표시로 인기도 파악

---

### 댓글/대댓글 영역

#### 구성 요소
- 댓글 리스트
  - 작성자 이름
  - 댓글 내용
  - 작성일
  - 대댓글 표시(들여쓰기 또는 연결선)
- 댓글 작성 입력 필드
- 댓글 작성 버튼

#### 위치
- 공감 영역 하단

#### 역할
- 게시글에 대한 의견 교환
- 대댓글을 통한 심화 토론 제공

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션
- (선택) 게시글 관련 보조 액션 영역
  - 예: 공유, 링크 복사 등(PRD 정의 시)

### 위치
- 화면 하단 고정 영역

### 역할
- 앱 내 주요 화면 이동 유지
- 게시글 소비 후 다음 행동 유도

---

## 게시글 상세 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 게시글 상세 | 상단/중앙 | 게시글 진입 | 게시글 데이터 로딩 |
| 2 | 게시글 상세 | 본문 영역 | 스크롤 | 추가 콘텐츠 표시 |
| 3 | 게시글 상세 | 공감 영역 | 공감 선택 | 공감 수 증가 |
| 4 | 게시글 상세 | 댓글 영역 | 댓글 작성/조회 | 댓글 추가/표시 |
| 5 | 게시글 상세 | 옵션 메뉴 | 수정/삭제 선택 | 권한 검증 후 처리 |
| 6 | 게시글 상세 | 뒤로 버튼 | 뒤로 이동 | 게시글 목록 복귀 |

---

## 설계 메모

- 게시글 상세 화면은 읽기 중심 화면이다.
- 댓글, 대댓글, 공감은 MVP에 포함된다(prd/07-release.md 기준).
- 수정/삭제는 명확한 권한 기반으로만 노출한다.
- 뒤로 이동 시 목록 상태(스크롤 위치)는 유지하는 것이 기본 원칙이다.
