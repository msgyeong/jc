# 공지사항 화면 텍스트 와이어프레임

<!-- [변경일: 2026-01-19] 변경 이유: 버전 표기 제거, 용어 통일, 댓글/대댓글/공감 기능 추가 -->

본 문서는 공지사항 기능을
- 목록
- 상세
- 작성(앱 내 관리자(직책 기반))
화면 단위로 분리하여 설계한다.

공지사항은 일반 게시글과 **성격이 명확히 구분**되며,
홈 화면 및 목록에서 **우선 노출 대상**이 된다.

---

# [공지사항 목록 화면]

## 화면 목적
- 전체 회원이 공지사항을 한곳에서 확인한다.
- 중요 공지(고정 공지)를 우선적으로 인지하게 한다.
- 공지 상세 화면으로의 진입점을 제공한다.

---

## 상단 영역

### 구성 요소
- 뒤로 버튼 또는 탭 내비게이션 연계
- 화면 제목: 공지사항

### 위치
- 화면 상단 고정 영역

### 역할
- 화면 맥락 명확화
- 이전 화면 또는 다른 탭으로 이동

---

## 중앙 영역 (스크롤 영역)

### 공지 리스트 영역

#### 구성 요소
- 공지 카드 리스트
  - 공지 제목
  - 등록일
  - 고정 표시(존재 시)
- 고정 공지 영역
  - 고정된 공지는 리스트 상단에 별도 또는 우선 배치

#### 위치
- 화면 중앙 스크롤 영역 전체

#### 역할
- 공지사항 탐색의 핵심 영역
- 최신 공지 및 중요 공지 식별 제공

---

### 빈 상태(Empty State)

#### 구성 요소
- 안내 문구
  - 예: "등록된 공지사항이 없습니다."

#### 역할
- 데이터 없음 상태에 대한 사용자 혼란 방지

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션
- 공지 작성 버튼(앱 내 관리자(직책 기반)에게만 노출)

### 위치
- 화면 하단 고정 영역

### 역할
- 주요 화면 이동
- 앱 내 관리자 공지 작성 진입 제공

---

## 공지사항 목록 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 공지 목록 | 리스트 영역 | 화면 진입 | 공지 목록 로딩 |
| 2 | 공지 목록 | 공지 카드 | 공지 선택 | 공지 상세 이동 |
| 3 | 공지 목록 | 작성 버튼 | 작성 선택 | 공지 작성 화면 이동 |

---

# [공지사항 상세 화면]

## 화면 목적
- 선택한 공지사항의 전체 내용을 확인한다.
- 공지의 중요도와 전달 내용을 명확히 인지한다.

---

## 상단 영역

### 구성 요소
- 뒤로 버튼
- 옵션 메뉴 버튼
  - 수정(앱 내 관리자(직책 기반))
  - 삭제(앱 내 관리자(직책 기반))

### 위치
- 화면 상단 고정 영역

### 역할
- 목록 화면으로 복귀
- 앱 내 관리자 관리 액션 진입

---

## 중앙 영역 (스크롤 영역)

### 공지 메타 정보 영역

#### 구성 요소
- 공지 제목
- 등록일
- 고정 여부 표시(존재 시)

#### 위치
- 중앙 영역 상단

#### 역할
- 공지의 핵심 정보 전달
- 중요 공지 식별

---

### 공지 본문 영역

#### 구성 요소
- 공지 내용 텍스트
- 줄바꿈 및 문단 유지

#### 위치
- 메타 정보 영역 하단

#### 역할
- 공지사항의 전체 내용 제공

---

### 참석자 조사 영역 (참석자 조사 활성화 시)

#### 구성 요소
- 참석자 조사 섹션 제목
- 참석 여부 선택 버튼 (참석/불참/미정)
- 참석자 통계 표시 (참석/불참/미정 수)
- 참석자 목록 표시 (이름, 참석 여부)

#### 위치
- 본문 영역 하단

#### 역할
- 공지사항 참석 여부 조사 기능 제공
- 참석자 정보 확인 기능 제공
- 참석 통계 확인 기능 제공

---

### 공감 영역

#### 구성 요소
- 공감 버튼 (아이콘 + 숫자)
- 공감한 사용자 목록(선택 시 표시)

#### 위치
- 참석자 조사 영역 하단 (또는 본문 하단)

#### 역할
- 공지에 대한 긍정 반응 제공

---

### 댓글/대댓글 영역

#### 구성 요소
- 댓글 리스트
  - 작성자 이름
  - 댓글 내용
  - 작성일
  - 대댓글 표시(들여쓰기 또는 연결선)
- 댓글 작성 입력 필드
- 댓글 작성 버튼

#### 위치
- 공감 영역 하단

#### 역할
- 공지에 대한 의견 교환
- 대댓글을 통한 심화 토론 제공

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션

### 위치
- 화면 하단 고정 영역

### 역할
- 다른 주요 화면으로 이동 유지

---

## 공지사항 상세 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 공지 상세 | 본문 영역 | 상세 진입 | 공지 데이터 로딩 |
| 2 | 공지 상세 | 참석자 조사 영역 | 참석 여부 선택 | 참석 여부 저장 및 통계 업데이트 |
| 3 | 공지 상세 | 공감 영역 | 공감 선택 | 공감 수 증가 |
| 4 | 공지 상세 | 댓글 영역 | 댓글 작성/조회 | 댓글 추가/표시 |
| 5 | 공지 상세 | 옵션 메뉴 | 수정/삭제 선택 | 권한 검증 |
| 6 | 공지 상세 | 뒤로 버튼 | 뒤로 이동 | 공지 목록 복귀 |

---

# [공지사항 작성 화면] (앱 내 관리자(직책 기반) 전용)

## 화면 목적
앱 내 관리자(직책 기반)가 새로운 공지사항을 작성하여 전체 회원에게 전달한다.
- 중요 공지를 설정할 수 있도록 한다.

---

## 상단 영역

### 구성 요소
- 취소 버튼
- 화면 제목: 공지 작성
- 등록 버튼

### 위치
- 화면 상단 고정 영역

### 역할
- 작성 흐름 제어
- 공지 등록 또는 작성 취소 제공

---

## 중앙 영역 (스크롤 영역)

### 제목 입력 영역

#### 구성 요소
- 공지 제목 입력 필드

#### 위치
- 중앙 영역 상단

#### 역할
- 공지 핵심 요약 입력

---

### 내용 입력 영역

#### 구성 요소
- 공지 내용 입력 필드
  - 여러 줄 텍스트 입력

#### 위치
- 제목 입력 영역 하단

#### 역할
- 공지 상세 내용 입력

---

### 중요 설정 영역

#### 구성 요소
- 고정 공지 설정 토글(PRD에 정의된 경우)

#### 위치
- 내용 입력 영역 하단

#### 역할
- 중요 공지 여부 설정

---

### 참석자 조사 설정 영역

#### 구성 요소
- 참석자 조사 활성화 옵션 (체크박스 또는 스위치)

#### 위치
- 중요 설정 영역 하단

#### 역할
- 참석자 조사 기능 활성화/비활성화 설정

---

## 하단 영역

### 구성 요소
- 입력 보조 영역(키보드 대응)

### 위치
- 화면 하단

### 역할
- 작성 환경 안정성 확보

---

## 공지사항 작성 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 공지 작성 | 제목 입력 | 제목 입력 | 입력값 반영 |
| 2 | 공지 작성 | 내용 입력 | 내용 입력 | 입력값 반영 |
| 3 | 공지 작성 | 중요 설정 | 고정 설정 | 설정값 반영 |
| 4 | 공지 작성 | 참석자 조사 설정 | 참석자 조사 활성화 선택 | 설정값 반영 |
| 5 | 공지 작성 | 등록 버튼 | 등록 선택 | 필수 입력 검증 |
| 6 | 공지 상세 | 결과 화면 | 등록 완료 | 공지 상세 이동 |

---

## 설계 메모

- 공지사항은 게시글과 시각적·구조적으로 구분되어야 한다.
- 고정 공지는 홈 및 목록에서 항상 우선 노출된다.
- 공지 작성 및 관리 기능은 앱 내 관리자(직책 기반)에게만 제공된다.
- 공지 상세는 읽기 전용이 기본이며, 관리 액션은 명확히 분리한다.
- 공지사항에 댓글/대댓글/공감 기능이 제공된다(prd/07-release.md 기준).
