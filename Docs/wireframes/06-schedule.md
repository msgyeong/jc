# 일정 화면 텍스트 와이어프레임

<!-- [변경일: 2026-01-19] 변경 이유: 버전 표기 제거, 용어 통일, 댓글/대댓글/공감 기능 추가 -->

본 문서는 일정 기능을
- 일정 목록/캘린더
- 일정 상세
- 일정 등록(앱 내 관리자(직책 기반))
화면 단위로 분리하여 설계한다.

일정은 공지 다음으로 중요한 정보이며,
홈 요약 → 일정 화면 → 상세 소비 흐름을 따른다.

---

# [일정 목록 / 캘린더 화면]

## 화면 목적
- 사용자가 전체 일정을 한눈에 파악한다.
- 날짜 기준으로 일정을 탐색한다.
- 특정 일정을 선택하여 상세 정보를 확인한다.

---

## 상단 영역

### 구성 요소
- 화면 제목: 일정
- 보기 전환 컨트롤(PRD 정의 시)
  - 월별 캘린더 / 목록 보기
- 오늘 이동 버튼(존재 시)

### 위치
- 화면 상단 고정 영역

### 역할
- 현재 화면 맥락 제공
- 일정 탐색 기준 전환

---

## 중앙 영역 (스크롤 영역)

### 캘린더 영역(기본 보기)

#### 구성 요소
- 월별 캘린더 그리드
  - 날짜 표시
  - 일정 존재 여부 표시(점/아이콘 등, 표현 방식은 설계만 정의)
- 선택된 날짜 강조 표시

#### 위치
- 중앙 영역 상단

#### 역할
- 날짜 기반 일정 탐색
- 일정이 있는 날짜를 직관적으로 인지

---

### 일정 리스트 영역

#### 구성 요소
- 선택된 날짜 또는 전체 일정 리스트
  - 일정 제목
  - 날짜
  - 시간(존재 시)
  - 장소(존재 시)

#### 위치
- 캘린더 영역 하단

#### 역할
- 선택 날짜의 일정 상세 목록 제공
- 일정 상세 화면으로의 진입 제공

---

### 빈 상태(Empty State)

#### 구성 요소
- 안내 문구
  - 예: "선택한 날짜에 일정이 없습니다."

#### 역할
- 일정 없음 상태 명확화

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션
- 일정 등록 버튼(앱 내 관리자(직책 기반)에게만 노출)

### 위치
- 화면 하단 고정 영역

### 역할
- 주요 화면 이동
- 앱 내 관리자 일정 등록 진입 제공

---

## 일정 목록 / 캘린더 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 일정 | 캘린더 | 화면 진입 | 일정 데이터 로딩 |
| 2 | 일정 | 날짜 선택 | 날짜 선택 | 해당 날짜 일정 표시 |
| 3 | 일정 | 일정 리스트 | 일정 선택 | 일정 상세 이동 |
| 4 | 일정 | 등록 버튼 | 등록 선택 | 일정 등록 화면 이동 |

---

# [일정 상세 화면]

## 화면 목적
- 선택한 일정의 상세 정보를 확인한다.
- 일정 참여 여부를 판단할 수 있는 정보를 제공한다.

---

## 상단 영역

### 구성 요소
- 뒤로 버튼
- 옵션 메뉴 버튼(앱 내 관리자(직책 기반))
  - 수정
  - 삭제

### 위치
- 화면 상단 고정 영역

### 역할
- 일정 목록/캘린더로 복귀
- 앱 내 관리자 일정 관리 액션 진입

---

## 중앙 영역 (스크롤 영역)

### 일정 메타 정보 영역

#### 구성 요소
- 일정 제목
- 일정 날짜
- 시간
- 장소

#### 위치
- 중앙 영역 상단

#### 역할
- 일정의 핵심 정보 전달

---

### 일정 상세 설명 영역

#### 구성 요소
- 일정 상세 설명 텍스트

#### 위치
- 메타 정보 영역 하단

#### 역할
- 일정 내용에 대한 추가 설명 제공

---

### 공감 영역

#### 구성 요소
- 공감 버튼 (아이콘 + 숫자)
- 공감한 사용자 목록(선택 시 표시)

#### 위치
- 상세 설명 영역 하단

#### 역할
- 일정에 대한 긍정 반응 제공

---

### 댓글/대댓글 영역

#### 구성 요소
- 댓글 리스트
  - 작성자 이름
  - 댓글 내용
  - 작성일
  - 대댓글 표시(들여쓰기 또는 연결선)
- 댓글 작성 입력 필드
- 댓글 작성 버튼

#### 위치
- 공감 영역 하단

#### 역할
- 일정에 대한 의견 교환
- 대댓글을 통한 심화 토론 제공

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션

### 위치
- 화면 하단 고정 영역

### 역할
- 앱 주요 화면 이동 유지

---

## 일정 상세 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 일정 상세 | 본문 영역 | 상세 진입 | 일정 데이터 로딩 |
| 2 | 일정 상세 | 공감 영역 | 공감 선택 | 공감 수 증가 |
| 3 | 일정 상세 | 댓글 영역 | 댓글 작성/조회 | 댓글 추가/표시 |
| 4 | 일정 상세 | 옵션 메뉴 | 수정/삭제 선택 | 권한 검증 |
| 5 | 일정 상세 | 뒤로 버튼 | 뒤로 이동 | 일정 화면 복귀 |

---

# [일정 등록 화면] (앱 내 관리자(직책 기반) 전용)

## 화면 목적
- 관리자가 새로운 일정을 등록한다.
- 필수 정보를 빠짐없이 입력하도록 유도한다.

---

## 상단 영역

### 구성 요소
- 취소 버튼
- 화면 제목: 일정 등록
- 등록 버튼

### 위치
- 화면 상단 고정 영역

### 역할
- 일정 등록 흐름 제어
- 등록 또는 취소 제공

---

## 중앙 영역 (스크롤 영역)

### 일정 제목 입력 영역

#### 구성 요소
- 일정 제목 입력 필드

#### 위치
- 중앙 영역 상단

#### 역할
- 일정 식별 정보 입력

---

### 날짜 선택 영역

#### 구성 요소
- 날짜 선택 UI

#### 위치
- 제목 입력 영역 하단

#### 역할
- 일정 날짜 지정

---

### 시간 입력 영역

#### 구성 요소
- 시작 시간 / 종료 시간 입력(PRD 정의 기준)

#### 위치
- 날짜 선택 영역 하단

#### 역할
- 일정 시간 정보 입력

---

### 장소 입력 영역

#### 구성 요소
- 장소 입력 필드

#### 위치
- 시간 입력 영역 하단

#### 역할
- 일정 장소 정보 입력

---

### 설명 입력 영역

#### 구성 요소
- 일정 설명 입력 필드
  - 여러 줄 텍스트 입력

#### 위치
- 장소 입력 영역 하단

#### 역할
- 일정 상세 설명 입력

---

## 하단 영역

### 구성 요소
- 입력 보조 영역(키보드 대응)

### 위치
- 화면 하단

### 역할
- 입력 흐름 안정성 확보

---

## 일정 등록 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 일정 등록 | 제목 입력 | 제목 입력 | 입력값 반영 |
| 2 | 일정 등록 | 날짜/시간 | 날짜·시간 선택 | 입력값 반영 |
| 3 | 일정 등록 | 장소/설명 | 내용 입력 | 입력값 반영 |
| 4 | 일정 등록 | 등록 버튼 | 등록 선택 | 필수 입력 검증 |
| 5 | 일정 상세 | 결과 화면 | 등록 완료 | 일정 상세 이동 |

---

## 설계 메모

- 일정은 날짜 중심 탐색이 기본이다.
- 일정 상세는 읽기 전용이 기본이며 관리 액션은 앱 내 관리자(직책 기반)에게만 제공된다.
- 홈 화면 요약 일정과 정보 구조를 일관되게 유지한다.
- 일정에 댓글/대댓글/공감 기능이 제공된다(prd/07-release.md 기준).
