# 회원 화면 텍스트 와이어프레임

<!-- [변경일: 2026-01-19] 변경 이유: 버전 표기 제거, 용어 통일(contributing.md 기준) -->

본 문서는 회원 기능을
- 회원 목록 / 검색
- 프로필 상세
화면 단위로 설계한다.

회원은 모두 동일한 사용자 경험을 가지며,
특우회 여부는 **프로필 정보로만 구분**된다.
기능/권한 분기의 기준이 아니다.

---

# [회원 목록 / 검색 화면]

## 화면 목적
- 사용자가 전체 회원을 탐색한다.
- 특정 회원을 빠르게 검색한다.
- 회원 프로필 상세 화면으로 진입한다.

---

## 상단 영역

### 구성 요소
- 화면 제목: 회원
- 검색 입력 필드
  - 이름 기준 검색
  - 기수/소속 등 PRD 정의 항목(존재 시)

### 위치
- 화면 상단 고정 영역

### 역할
- 회원 탐색 범위 축소
- 현재 화면 맥락 제공

---

## 중앙 영역 (스크롤 영역)

### 회원 리스트 영역

#### 구성 요소
- 회원 카드 리스트
  - 이름
  - 기수 또는 소속 정보
  - 특우회 여부 표시(텍스트 또는 배지)
- 회원 카드 전체 탭 가능 영역

#### 위치
- 화면 중앙 스크롤 영역 전체

#### 역할
- 회원 탐색의 핵심 영역
- 프로필 상세 화면으로의 진입 제공

---

### 빈 상태(Empty State)

#### 구성 요소
- 안내 문구
  - 예: "검색 결과가 없습니다."

#### 역할
- 검색 결과 없음 상태 명확화

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션

### 위치
- 화면 하단 고정 영역

### 역할
- 주요 화면 간 이동 유지

---

## 회원 목록 / 검색 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 회원 목록 | 검색 입력 | 검색어 입력 | 회원 목록 필터링 |
| 2 | 회원 목록 | 회원 카드 | 회원 선택 | 프로필 상세 이동 |

---

# [프로필 상세 화면]

## 화면 목적
- 선택한 회원의 기본 정보를 확인한다.
- 회원의 소속 및 특우회 여부를 명확히 인지한다.
- 회원 관리(상태/권한/승인)는 **관리자 웹**에서만 수행한다(최고 관리자).

---

## 상단 영역

### 구성 요소
- 뒤로 버튼
- 옵션 메뉴 버튼(없음)

### 위치
- 화면 상단 고정 영역

### 역할
- 회원 목록 화면으로 복귀
- (참고) 회원 관리 액션은 관리자 웹에서만 제공

---

## 중앙 영역 (스크롤 영역)

### 기본 정보 영역

#### 구성 요소
- 회원 이름
- 기수 또는 소속
- 특우회 여부 표시
- 기타 PRD 정의 기본 정보

#### 위치
- 중앙 영역 상단

#### 역할
- 회원 정체성 및 소속 정보 전달

---

### 추가 정보 영역(PRD 정의 시)

#### 구성 요소
- 연락 수단(존재 시)
- 기타 공개 정보(PRD 기준)

#### 위치
- 기본 정보 영역 하단

#### 역할
- 회원 간 커뮤니케이션 보조

---

## 하단 영역

### 구성 요소
- 하단 탭 내비게이션

### 위치
- 화면 하단 고정 영역

### 역할
- 앱 주요 화면 이동 유지

---

## 프로필 상세 화면 유저 플로우 표

| Step | 화면명 | UI 구성 | 사용자 행동 | 시스템 동작 |
|---:|---|---|---|---|
| 1 | 프로필 상세 | 정보 영역 | 상세 진입 | 회원 정보 로딩 |
| 2 | 프로필 상세 | 뒤로 버튼 | 뒤로 이동 | 회원 목록 복귀 |

---

## 설계 메모

- 특우회 여부는 단순 정보 표시 항목이다.
- 회원 간 기능 차별은 존재하지 않는다.
- 회원 관리 기능은 앱에 노출되지 않으며, 관리자 웹에서만 제공한다(최고 관리자 수행).
- 프로필 상세는 읽기 중심 화면으로 유지한다.
