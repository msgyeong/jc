# 앱 화면 목록 + 화면 ID 확정

<!-- [변경일: 2026-01-19] 변경 이유: 버전 표기 제거, 용어 통일(contributing.md 기준) -->

본 문서는 설계된 **앱 화면**에 대해,
- 핵심 유저 플로우
- 주요 기능 상세 설계
- 화면별 와이어프레임
을 기준으로 **중복 없이 화면을 확정**한다.

> **참고**: 관리자 웹 화면은 별도 관리한다.

원칙:
- 하나의 화면은 하나의 주요 목적을 가진다.
- 앱 내 관리자(직책 기반) 전용 화면은 별도 표기한다.
- 향후 확장 가능 화면은 선택 항목으로 분리한다.

- [x] 모든 유저 플로우는 최소 1개 이상의 화면과 연결됨
- [x] 홈 → 목록 → 상세 → 작성 구조 일관성 유지
- [x] 앱 내 관리자(직책 기반) 기능은 일반 화면과 명확히 분리됨

## 6. 최종 검증 결과

- 모든 화면은 최소 1개 이상의 기능과 연결됨
- 모든 기능은 최소 1개 이상의 화면과 연결됨
- 모든 유저 플로우는 실제 화면 흐름으로 구현 가능
- 앱 내 관리자(직책 기반) 전용 기능은 화면 단위로 완전히 분리됨
- 특우회 여부는 프로필 정보에서만 사용됨
- 게시글/공지/일정 공통 기능(댓글/대댓글/공감)이 MVP에 포함됨
