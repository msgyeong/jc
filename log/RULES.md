# 로그 작성 규칙

이 폴더는 공동개발자 간 작업 내역 공유용입니다.

## 파일명 규칙

- 형식: `[YYYY-MM-DD][작성자].md`
- 예시:
  - `[2026-02-15][msgyeong].md`
  - `[2026-02-16][devkim].md`

## 작성 규칙

1. 하루 1파일을 기본으로 작성합니다.
2. 같은 날 추가 작업은 같은 파일에 시간 구분 섹션을 추가합니다.
3. 아래 템플릿 순서를 유지합니다.

## 템플릿

```md
# 작업 로그 - YYYY-MM-DD (작성자)

## 작업 요약
- 핵심 작업 2~5줄

## 변경 파일
- 경로

## 상세 작업
### 1) 작업 항목
- 무엇을 왜 변경했는지

## 테스트/확인
- 실행한 테스트와 결과

## 이슈/메모
- 남은 작업, 주의사항, 협업 요청사항
```

## 협업 규칙

- 다른 사람 로그는 수정하지 않습니다.
- 본인 파일만 수정/추가합니다.
- 필요한 경우 마지막에 `@멘션` 형태로 요청사항을 남깁니다.
